# CIFAR-100 分类项目讲解与算法详解

## 一、项目整体概览

本项目的任务是**用深度学习给图片分类**——在 CIFAR-100 数据集上，分别用 **MLP（多层感知机）** 和 **CNN（卷积神经网络）** 两种模型，搭配不同的**学习率**和**优化器**，跑一批对比实验，最终看谁的分类效果好、为什么好。

整个项目可以理解成一个**实验工厂**：给它指定"用什么模型 × 什么优化器 × 什么学习率"的组合，它就自动帮你跑完所有实验、保存曲线图、保存模型、生成汇总表格。

### 文件一览

| 文件 | 作用 |
|------|------|
| `src/models.py` | 定义 MLP 和 CNN 两个模型的网络结构 |
| `src/trainer.py` | 数据加载、优化器构建、训练循环、画图、工具函数 |
| `src/__init__.py` | 统一导出，方便 `train.py` 一行导入 |
| `train.py` | 主入口，遍历所有实验组合并逐个运行 |
| `summarize_results.py` | 读取所有实验结果，生成汇总表格和自动报告 |
| `reports/实验报告.md` | 半成品实验报告模板，跑完实验后填入结果即可 |

## 二、数据集：CIFAR-100

CIFAR-100 是经典的小图像数据集：

- 每张图 **32×32 像素，3 个颜色通道**（RGB）
- 一共 **100 个类别**（苹果、汽车、老虎……非常细粒度）
- 训练集 50000 张，测试集 10000 张

因为图片很小、类别又多，所以分类难度其实不低。

### 数据预处理

代码里做了两套预处理方案：

- **训练集**：随机裁剪（`RandomCrop`）+ 随机水平翻转（`RandomHorizontalFlip`）+ 标准化（`Normalize`）。随机裁剪和翻转是**数据增强**手段——让模型见到更多变化过的样本，防止死记硬背（过拟合）。
- **验证集 / 测试集**：只做标准化，不做随机变换，保证评估结果稳定。

标准化用的是 CIFAR-100 的全局均值 `(0.5071, 0.4867, 0.4408)` 和标准差 `(0.2675, 0.2565, 0.2761)`，把像素值拉到均值 0、标准差 1 附近，有利于梯度的稳定传播。

## 三、模型一：MLP（多层感知机）

### 网络结构

```text
输入图片 32×32×3
    ↓
Flatten：拍平成长度 3072 的一维向量
    ↓
Linear(3072 → 1024) + ReLU + Dropout(0.3)
    ↓
Linear(1024 → 512)  + ReLU + Dropout(0.3)
    ↓
Linear(512 → 100)：输出 100 个类别的得分
```

### 通俗解释

MLP 是最朴素的神经网络。它的做法很简单——把一张 32×32×3 的彩色图片**拍平**成一个长度为 3072 的一维向量，然后依次通过两个全连接层（1024 → 512 → 100），最后输出 100 个数，代表属于每个类别的"打分"。

### 关键组件

| 组件 | 作用 | 直觉理解 |
|------|------|---------|
| **Flatten** | 把二维图片拍成一维向量 | 就像把一张照片裁成纸条 |
| **Linear** | 全连接层，每个输入与每个输出全部相连 | 把所有信息打碎重新组合 |
| **ReLU** | 激活函数，把负数变 0，正数保留 | 没有它，再多全连接层叠起来也等价于一层（线性变换的复合还是线性变换） |
| **Dropout(0.3)** | 训练时随机关掉 30% 的神经元 | 防止网络过度依赖某几个神经元，提高泛化能力 |

### MLP 的局限

它把图片拍平后，**丢掉了所有空间位置信息**。对它来说，左上角的像素和右下角的像素没有"邻居"关系，这在图像任务里是很大的劣势。

## 四、模型二：CNN（卷积神经网络）

### 网络结构

```text
输入图片 32×32×3
    ↓
─── 第一阶段 ───
Conv2d(3→64, 3×3) + BatchNorm + ReLU
Conv2d(64→64, 3×3) + ReLU
MaxPool2d(2×2) + Dropout2d(0.3)
→ 输出：64×16×16
    ↓
─── 第二阶段 ───
Conv2d(64→128, 3×3) + BatchNorm + ReLU
Conv2d(128→128, 3×3) + ReLU
MaxPool2d(2×2) + Dropout2d(0.3)
→ 输出：128×8×8
    ↓
─── 第三阶段 ───
Conv2d(128→256, 3×3) + BatchNorm + ReLU
MaxPool2d(2×2)
→ 输出：256×4×4
    ↓
─── 分类头 ───
Flatten → Linear(4096→512) + ReLU + Dropout(0.3) → Linear(512→100)
```

### 通俗解释

CNN 和 MLP 的核心区别在于——**它不把图片拍平，而是用小窗口在图片上滑动扫描，逐步提取局部特征**。

整个网络分两大块：

**特征提取部分**：网络分为三个"阶段"，每个阶段都是"卷积 → 激活 → 池化"的套路。

| 阶段 | 通道数变化 | 空间尺寸变化 | 做了什么 |
|------|-----------|-------------|---------|
| 第一阶段 | 3 → 64 | 32×32 → 16×16 | 两层卷积 + 一次池化 |
| 第二阶段 | 64 → 128 | 16×16 → 8×8 | 两层卷积 + 一次池化 |
| 第三阶段 | 128 → 256 | 8×8 → 4×4 | 一层卷积 + 一次池化 |

最终输出的是一个 256×4×4 = 4096 维的特征图。

**分类头部分**：特征提取完之后，把 256×4×4 的特征图拍平成 4096 维向量，通过两个全连接层（4096 → 512 → 100）输出分类结果。

### 关键组件详解

| 组件 | 作用 | 直觉理解 |
|------|------|---------|
| **Conv2d(3, 64, 3×3, padding=1)** | 3×3 卷积核在图片上滑动 | 一个"放大镜"，每次只看 3×3 的小区域，提取边缘、颜色变化等局部模式。从 3 通道（RGB）变成 64 通道，意味着学了 64 种不同的"观察角度" |
| **BatchNorm2d(64)** | 批归一化，把每层输出拉回均值 0、方差 1 附近 | 每层之间加个"校准器"，让梯度更稳定、训练更快 |
| **MaxPool2d(2×2)** | 最大池化，保留 2×2 区域里的最大值 | 相当于"缩略图"——尺寸缩小一半，信息变少但关键内容还在 |
| **Dropout2d(0.3)** | 随机关掉 30% 的**整个特征通道** | 比普通 Dropout 更适合卷积层，防止过拟合 |

### CNN 为什么比 MLP 好

卷积操作天然保留了图像的**空间局部性**——相邻像素会被一起处理，能捕捉到边缘、纹理、形状等图案；再通过池化逐步"缩放"，高层特征就能看到更大范围的语义信息。MLP 把图片拍平后这些信息全丢了。

## 五、损失函数：交叉熵（CrossEntropyLoss）

分类问题的标配损失函数。直觉上理解：

1. 模型对每张图输出 100 个数（logits），代表"认为是每个类别的程度"
2. 交叉熵做的事：看你**给正确类别打了多高的分**。如果正确类别得分很高，损失就小；如果正确类别得分低（其他类别抢了分数），损失就大
3. 训练的目标就是**把损失降到最低**

数学公式：

$$L = -\log\frac{e^{z_y}}{\sum_{j=1}^{100} e^{z_j}}$$

其中 $z_y$ 是正确类别的 logit 值。分子越大（正确类别得分越高），损失越小。

## 六、优化器：SGD vs Adam

优化器决定了**模型参数怎么更新**。可以用一个比喻来理解——你站在山上，想走到最低点（损失最小）：

### SGD（随机梯度下降 + 动量）

- **基本思路**：每一步沿着当前位置最陡的下坡方向走一步，步长就是**学习率**
- **动量（momentum=0.9）**：不只看当前这一步的坡度，还"记住"之前的行进方向。就像一个滚下山的球，有惯性，不容易被小坑小洼拦住
- **特点**：对学习率比较敏感——太大会震荡飞出去，太小会爬得很慢。但调好后泛化性能往往不错

更新公式：

$$v_t = \mu \cdot v_{t-1} + \nabla L(\theta_{t-1})$$
$$\theta_t = \theta_{t-1} - \alpha \cdot v_t$$

其中 $\alpha$ 是学习率，$\mu$ 是动量系数。

### Adam（自适应矩估计）

- **基本思路**：在 SGD 基础上更"聪明"——它会**自动为每个参数调整学习率**。对于更新频繁的参数，步子迈小点；对于更新稀少的参数，步子迈大点
- **内部维护两个量**：一阶矩（梯度的移动平均，类似动量）和二阶矩（梯度平方的移动平均，用于自适应缩放步长）
- **特点**：几乎不需要精心调学习率，开箱即用，收敛通常更快。但在某些场景下泛化可能略逊于调好的 SGD

#### 深入理解"自动为每个参数调整学习率"

用一个生活比喻来解释：

> 假设你管理一家公司，公司有 1000 个员工（= 1000 个参数）。SGD 的方式是**给所有人发一样的薪资调整额**（统一学习率）。但实际上每个员工的情况不同——有人工作量波动剧烈（梯度忽大忽小），有人一直很稳定（梯度大小稳定）。
>
> Adam 的方式是**给每个员工单独建档**，记录他最近的工作量（一阶矩）和工作量的波动程度（二阶矩），然后：
> - 工作量波动大的员工 → 调整幅度保守一点（除以一个大数），避免过度反应
> - 工作量一直稳定的员工 → 调整幅度可以大一点（除以一个小数），因为方向很确定

具体到公式，Adam 为**每一个参数**维护两个变量：

$$m_t = \beta_1 \cdot m_{t-1} + (1 - \beta_1) \cdot g_t$$
$$v_t = \beta_2 \cdot v_{t-1} + (1 - \beta_2) \cdot g_t^2$$

| 变量 | 含义 | 直觉 |
|------|------|------|
| $g_t$ | 当前这一步的梯度 | 这次要调多少 |
| $m_t$（一阶矩） | 梯度的**移动平均** | 最近一段时间的"平均调整方向"，相当于动量 |
| $v_t$（二阶矩） | 梯度**平方**的移动平均 | 最近一段时间梯度的"波动剧烈程度" |
| $\beta_1 = 0.9$ | 一阶矩的衰减系数 | 记住最近 ~10 步的平均梯度 |
| $\beta_2 = 0.999$ | 二阶矩的衰减系数 | 记住最近 ~1000 步的波动情况 |

然后做**偏差校正**（因为 m 和 v 初始为 0，前几步估计值偏小）：

$$\hat{m}_t = \frac{m_t}{1 - \beta_1^t}, \quad \hat{v}_t = \frac{v_t}{1 - \beta_2^t}$$

最终更新参数：

$$\theta_t = \theta_{t-1} - \alpha \cdot \frac{\hat{m}_t}{\sqrt{\hat{v}_t} + \epsilon}$$

**关键在分母** $\sqrt{\hat{v}_t}$：

- 如果某个参数的梯度一直很大且波动剧烈 → $v_t$ 大 → $\sqrt{v_t}$ 大 → **实际步长被缩小**（谨慎调整）
- 如果某个参数的梯度一直很小且稳定 → $v_t$ 小 → $\sqrt{v_t}$ 小 → **实际步长被放大**（大胆调整）

这就实现了"每个参数有自己的学习率"。

#### SGD vs Adam 对比总结

| | SGD + Momentum | Adam |
|---|---|---|
| 学习率 | 所有参数共用一个 | 每个参数自动调整 |
| 需要调参吗 | 需要精心选学习率 | 开箱即用（默认 lr=1e-3） |
| 收敛速度 | 较慢，但稳扎稳打 | 较快 |
| 泛化性能 | 调好后往往更好 | 有时略逊 |
| 内存开销 | 每个参数多存 1 个状态（动量） | 每个参数多存 2 个状态（m 和 v） |
| 适用场景 | 追求极致性能、愿意调参 | 快速实验、不想花时间调参 |

### Weight Decay（权重衰减）

两个优化器都加了 `weight_decay=1e-4`，它的作用是每次更新时**把参数往零的方向拉一点点**，等价于给损失函数加了一个 L2 正则项。防止参数变得过大，是另一种防过拟合手段。

## 七、学习率（Learning Rate）

实验中对比了三个学习率：**0.1、0.01、0.001**。

学习率控制的是每一步参数更新的"步幅"：

| 学习率 | 表现预期 | 比喻 |
|--------|---------|------|
| **0.1（太大）** | 走得太猛，容易在最优点附近来回震荡，甚至越走越远（loss 不降反升） | 下山时跳着走，可能一脚踩过头跳到对面山坡上 |
| **0.001（太小）** | 每步都像蚂蚁爬，收敛极慢，10 个 epoch 可能还没走到好的位置 | 下山时一毫米一毫米地挪 |
| **0.01（适中）** | 通常是比较好的起点，能在合理时间内稳定收敛 | 正常步伐走路，稳定下山 |

这也是为什么作业要求对比不同学习率——让你亲眼看到"过大震荡 / 过小缓慢 / 适中收敛"的现象。

## 八、训练流程详解

### 单个 epoch 的流程

每个 epoch 做两件事：

1. **训练阶段**（`model.train()`）：遍历训练集的每个 batch → 前向传播算 loss → 反向传播算梯度 → 优化器更新参数
2. **验证阶段**（`model.eval()`）：遍历验证集的每个 batch → 只算 loss 和准确率，**不更新参数**

同时跟踪**验证集上表现最好的那一轮**的模型参数（Early Stopping 的思想：不一定是最后一轮最好，可能中间某轮泛化最优）。训练结束后，用最好的参数在测试集上做最终评估。

### 实验矩阵

`train.py` 的核心思路是**笛卡尔积遍历**所有组合：

```text
模型: [mlp, cnn] × 优化器: [sgd, adam] × 学习率: [0.1, 0.01, 0.001]
= 2 × 2 × 3 = 12 组实验
```

每组实验自动保存：

- **loss 曲线图**（`loss_curve.png`）：训练 loss / 验证 loss 和训练 accuracy / 验证 accuracy 随 epoch 的变化
- **预测可视化**（`prediction_examples.png`）：从测试集取样展示模型预测结果 vs 真实标签
- **模型参数**（`best_model.pt`）和**指标 JSON**（`metrics.json`）

## 九、关键概念速查表

| 概念 | 一句话解释 |
|------|-----------|
| **前向传播** | 数据从输入层经过各层计算，最终得到输出（预测值） |
| **反向传播** | 根据损失函数，从后往前计算每个参数的梯度（该往哪个方向调、调多少） |
| **梯度** | 损失函数对参数的导数，告诉你"往哪走损失能变小" |
| **epoch** | 完整遍历一遍训练集 |
| **batch** | 一次送入模型的一小批样本（这里 128 张），用小批量近似整体梯度 |
| **过拟合** | 模型在训练集上表现很好但在新数据上很差——"背答案"了 |
| **正则化** | 限制模型复杂度的手段（Dropout、Weight Decay 等），防止过拟合 |
| **数据增强** | 对训练图片做随机变换（裁剪、翻转等），让模型见更多样本变体 |
| **logits** | 模型输出的原始得分（未经 softmax 归一化），值越大表示越倾向该类别 |
| **softmax** | 把 logits 转换成概率分布（所有值变为正数且加和为 1） |

## 十、实验预期结论

根据算法原理，你跑完实验应该能观察到以下规律：

1. **CNN 显著优于 MLP**：因为卷积结构天然适合提取图像的空间局部特征，而 MLP 丢掉了所有位置信息
2. **学习率 0.1 在 Adam 上容易震荡**：Adam 自带自适应步长，再给大学习率会步子过猛
3. **学习率 0.001 在 SGD 上收敛缓慢**：SGD 没有自适应机制，小学习率就真的走得慢
4. **Adam + 0.001 通常效果稳定**：Adam 在中小学习率下开箱即用
5. **SGD + 0.01 + 动量** 可能泛化不错：经典调参组合，收敛稍慢但最终结果可能与 Adam 持平甚至更好
6. **某些细粒度类别容易混淆**：比如不同品种的花、不同类型的车辆，在 32×32 的低分辨率下本来就很难区分
