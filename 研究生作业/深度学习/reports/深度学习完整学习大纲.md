# 第一部分 基础层：地基

> 这部分是所有后续内容的前置知识。不需要精通，但至少要"看到公式不慌"。

## 1. 数学基础

### 1.1 线性代数

深度学习的核心运算就是**矩阵乘法**。你需要理解：

| 概念 | 为什么重要 | 直觉 |
|------|-----------|------|
| 向量 | 一个样本的特征就是一个向量 | 一组有序的数字 |
| 矩阵 | 一批样本 = 矩阵，网络权重 = 矩阵 | 数字排成的表格 |
| 矩阵乘法 | 全连接层的本质就是矩阵乘法 | 一种"信息混合"操作 |
| 转置 | 数据维度变换时常用 | 行列互换 |
| 广播 | NumPy/PyTorch 里形状不同的张量怎么相加 | 自动扩展小矩阵去匹配大矩阵 |

你不需要会证明特征值分解，但需要**能手算一个 2×3 矩阵乘 3×2 矩阵**。

### 1.2 微积分

反向传播的数学本质是**链式法则**。你需要理解：

| 概念 | 为什么重要 |
|------|-----------|
| 导数 | "这个参数变一点点，损失会变多少" |
| 偏导数 | 多变量函数对其中一个变量求导 |
| 链式法则 | 复合函数的导数 = 各层导数相乘。反向传播就是在自动执行链式法则 |
| 梯度 | 所有偏导数组成的向量，指向"损失上升最快的方向"，取反就是下降最快的方向 |

**核心直觉**：梯度告诉你"参数该往哪个方向调"，学习率告诉你"每次调多大一步"。

### 1.3 概率与统计

| 概念 | 在深度学习里的角色 |
|------|------------------|
| 概率分布 | softmax 输出的就是一个概率分布 |
| 条件概率 | "在看到这张图的条件下，它是猫的概率" |
| 最大似然估计 | 交叉熵损失的理论来源 |
| 期望与方差 | BatchNorm 在做的事 |
| 贝叶斯定理 | 理解某些高级方法（变分自编码器等）的基础 |

## 2. 编程工具

### 2.1 Python 基础

- 列表、字典、函数、类
- NumPy：矩阵运算库，所有深度学习框架的底层都在做类似的事
- Matplotlib：画训练曲线、可视化结果

### 2.2 深度学习框架

当前主流是 **PyTorch**（学术界和工业界都在用），其次是 TensorFlow/Keras。

PyTorch 的核心概念：

| 概念 | 作用 |
|------|------|
| `torch.Tensor` | 多维数组，类似 NumPy 的 ndarray，但支持 GPU 加速和自动求导 |
| `nn.Module` | 所有网络层和模型的基类，定义 `forward()` 方法 |
| `autograd` | 自动微分引擎，你只管写前向计算，它自动帮你算梯度 |
| `DataLoader` | 数据加载器，自动分批、打乱、多进程预取 |
| `optimizer` | 优化器，根据梯度更新参数 |
| `.to(device)` | 把数据或模型搬到 GPU/CPU 上 |

# 第二部分 核心层：所有神经网络的共同原理

> 不管是 CNN、RNN 还是 Transformer，它们都共享以下这套"基础设施"。

## 3. 神经元与网络结构

### 3.1 单个神经元

```text
输入 x₁ ──→ ×w₁ ──┐
输入 x₂ ──→ ×w₂ ──┼──→ 求和 + 偏置 b ──→ 激活函数 f ──→ 输出 y
输入 x₃ ──→ ×w₃ ──┘

y = f(w₁x₁ + w₂x₂ + w₃x₃ + b)
```

一个神经元做的事：**加权求和 → 加偏置 → 过激活函数**。

### 3.2 层（Layer）

把很多神经元排成一排，就是一层。常见的层类型：

| 层类型 | 做什么 | 典型用途 |
|--------|--------|---------|
| 全连接层（Linear） | 每个输入连接每个输出 | 分类头、MLP |
| 卷积层（Conv） | 用小窗口扫描提取局部特征 | 图像处理 |
| 循环层（RNN/LSTM/GRU） | 按时间步依次处理序列 | 文本、时间序列 |
| 注意力层（Attention） | 让序列中的每个位置"关注"其他位置 | Transformer |
| 归一化层（BatchNorm/LayerNorm） | 稳定每层输出的分布 | 几乎所有网络都用 |
| Dropout 层 | 随机关闭神经元防过拟合 | 训练阶段 |

### 3.3 网络 = 层的组合

```text
输入 → 层1 → 层2 → 层3 → ... → 层N → 输出
```

深度学习里的"深度"就是指**层数多**。层数越多，网络能表达的函数就越复杂——但也越难训练。

## 4. 激活函数

激活函数的作用：引入非线性。没有它，再多层也等于一层。

| 函数 | 公式 | 特点 | 现状 |
|------|------|------|------|
| **Sigmoid** | 1/(1+e⁻ˣ) | 输出在 (0,1)，可解释为概率 | 几乎只在输出层用（二分类） |
| **Tanh** | (eˣ-e⁻ˣ)/(eˣ+e⁻ˣ) | 输出在 (-1,1)，以 0 为中心 | 早期 RNN 常用 |
| **ReLU** | max(0, x) | 简单高效，梯度不消失 | 最常用的默认选择 |
| **Leaky ReLU** | x>0 时 x，x≤0 时 0.01x | 解决 ReLU 的"死神经元"问题 | ReLU 的常用替代 |
| **GELU** | x·Φ(x) | 平滑版 ReLU，Transformer 里的标配 | GPT、BERT 等都在用 |
| **Swish/SiLU** | x·sigmoid(x) | 谷歌搜索出的激活函数 | 部分视觉模型在用 |

**选择建议**：新手默认用 ReLU。做 Transformer 用 GELU。做二分类输出层用 Sigmoid。做多分类输出层用 Softmax。

## 5. 损失函数

损失函数衡量"模型预测和真实答案差多远"，训练的目标就是把损失降到最低。

| 损失函数 | 适用场景 | 直觉 |
|---------|---------|------|
| **MSE（均方误差）** | 回归问题 | 预测值和真实值的差的平方的平均 |
| **交叉熵（CrossEntropy）** | 分类问题 | "你给正确类别打了多少分" |
| **二元交叉熵（BCE）** | 二分类 / 多标签 | 每个类别独立判断是/否 |
| **Focal Loss** | 类别不均衡的分类 | 给难样本更大的权重 |
| **对比损失（Contrastive）** | 表示学习 | 让相似的靠近、不同的远离 |

## 6. 优化器

优化器决定**怎么根据梯度更新参数**。

| 优化器 | 核心思想 | 特点 |
|--------|---------|------|
| **SGD** | 沿梯度方向走一步 | 最朴素，但调好了效果往往最好 |
| **SGD + Momentum** | 加"惯性"，不容易被小坑困住 | 工业界常用组合 |
| **Adam** | 自适应调整每个参数的学习率 | 开箱即用，收敛快 |
| **AdamW** | Adam + 正确的权重衰减 | 当前最推荐的默认优化器 |
| **LAMB/LARS** | 大 batch 训练专用 | 分布式训练场景 |

**选择建议**：不确定就用 AdamW。追求极致性能可以试 SGD + Momentum + 学习率调度。

## 7. 学习率与调度

学习率可能是**最重要的超参数**。

### 7.1 学习率调度策略

| 策略 | 做法 | 适用场景 |
|------|------|---------|
| **固定学习率** | 始终不变 | 快速实验 |
| **StepLR** | 每隔 N 个 epoch 乘以 0.1 | 经典做法 |
| **CosineAnnealing** | 按余弦曲线从大到小衰减 | 当前主流 |
| **Warmup + Cosine** | 先花几个 epoch 从 0 升到目标值，再余弦衰减 | Transformer 训练标配 |
| **ReduceOnPlateau** | 验证集指标不再提升时自动降低 | 不想手动调的时候 |

### 7.2 学习率的直觉

```text
学习率太大：    ╱╲╱╲╱╲╱╲   震荡，不收敛
学习率适中：    ╲                稳定下降
                 ╲
                  ╲____    收敛到好的位置
学习率太小：    ─────── ... 收敛极慢，可能困在局部
```

## 8. 正则化——防止过拟合

| 方法 | 做法 | 直觉 |
|------|------|------|
| **L2 正则 / Weight Decay** | 参数越大，额外惩罚越大 | 不让参数"疯长" |
| **Dropout** | 训练时随机关掉部分神经元 | 迫使网络不依赖任何单个神经元 |
| **数据增强** | 对训练数据做随机变换 | 让网络见更多"变体" |
| **Early Stopping** | 验证集指标不再提升就停止训练 | 别训过头了 |
| **BatchNorm** | 标准化每层输出 | 附带轻微正则化效果 |
| **Label Smoothing** | 把标签从 [0,1] 变成 [0.05, 0.95] | 防止模型对预测"过度自信" |
| **Mixup / CutMix** | 混合两张训练图和它们的标签 | 高级数据增强 |

## 9. 反向传播与计算图

### 9.1 计算图

PyTorch 在你做前向计算时，会偷偷记录所有操作，构建一棵"计算图"：

```text
x → Linear → ReLU → Linear → ReLU → Linear → loss
```

每个箭头都是一个可微分的操作。

### 9.2 反向传播

调用 `loss.backward()` 时，PyTorch 从 loss 出发，沿着计算图**反向**遍历，用链式法则逐层算出每个参数的梯度。

```text
前向：x → a → b → c → loss
反向：x ← a ← b ← c ← loss    （梯度从后往前传）
```

你**不需要手写任何求导代码**，框架全部自动完成。

# 第三部分 架构层：各种网络结构

> 这是深度学习最丰富多彩的部分。不同的任务催生了不同的网络架构。

## 10. CNN（卷积神经网络）—— 擅长看图

适用于：图像分类、目标检测、图像分割、视频分析

### 核心组件

- **卷积层**：用小窗口提取局部特征
- **池化层**：缩小尺寸，保留关键信息
- **BatchNorm**：稳定训练
- **全连接层**：最终分类

### 经典网络演进

```text
LeNet (1998)     最早的CNN，识别手写数字
    ↓
AlexNet (2012)   深度CNN崛起，ImageNet冠军，引入ReLU和Dropout
    ↓
VGGNet (2014)    全用3×3小卷积核，靠深度取胜（16-19层）
    ↓
GoogLeNet (2014) Inception模块：并行使用不同大小的卷积核
    ↓
ResNet (2015)    残差连接：解决了"网络越深反而越差"的退化问题
    ↓
DenseNet (2017)  每一层都和前面所有层相连
    ↓
EfficientNet (2019) 同时优化深度、宽度和分辨率
    ↓
ConvNeXt (2022)  用Transformer的设计理念改造CNN，证明CNN还能更强
```

### 关键创新：残差连接（ResNet）

```text
普通网络：    x → 卷积 → 卷积 → 输出
残差网络：    x → 卷积 → 卷积 → (+x) → 输出
                                  ↑
                              跳跃连接
```

意义：如果后面的层学不到有用的东西，至少可以直接把输入传过去（相当于"什么都不做"），不会比浅层网络更差。这让训练几百层的网络成为可能。

> 关于 CNN 的详细教程，参见同目录下的《CNN从零开始教程.md》。

## 11. RNN 家族（循环神经网络）—— 擅长处理序列

适用于：文本、时间序列、语音（但现在很多场景已被 Transformer 取代）

### 11.1 基本 RNN

```text
时刻1          时刻2          时刻3
 x₁             x₂             x₃
  ↓              ↓              ↓
┌────┐  h₁   ┌────┐  h₂   ┌────┐  h₃
│ RNN │──────→│ RNN │──────→│ RNN │──────→
└────┘        └────┘        └────┘
```

核心思想：**带"记忆"的网络**。每个时刻，RNN 接收当前输入和上一时刻的隐状态，输出新的隐状态。隐状态就像一个"备忘录"，记录了之前看到的信息。

问题：时间步越多，早期的信息越容易被"遗忘"（梯度消失）。

### 11.2 LSTM（长短时记忆网络）

```text
┌────────────────────────────────────┐
│              LSTM 单元              │
│                                    │
│  遗忘门：决定丢掉多少旧记忆         │
│  输入门：决定写入多少新信息          │
│  输出门：决定输出多少当前记忆        │
│                                    │
│  记忆单元 C：长期记忆的"高速公路"    │
└────────────────────────────────────┘
```

LSTM 通过三个"门"来控制信息的流入流出，解决了普通 RNN 的长期记忆问题。

### 11.3 GRU（门控循环单元）

LSTM 的简化版，只有两个门（重置门和更新门），参数更少，效果通常和 LSTM 差不多。

### 11.4 RNN 的局限

- **无法并行**：必须按时间步顺序计算，GPU 利用率低
- **长距离依赖仍然困难**：即使是 LSTM，几千步之前的信息也很难保留
- **被 Transformer 取代**：在大多数 NLP 任务上，Transformer 已经全面超越 RNN

## 12. Transformer —— 当前的"万能架构"

适用于：几乎所有领域（NLP、CV、语音、多模态……）

### 12.1 核心创新：自注意力（Self-Attention）

```text
"小猫坐在垫子上，它很开心"

处理"它"这个词时：
  - 和"小猫"的关联度 = 0.8  ← 注意力集中在这里
  - 和"坐在"的关联度 = 0.05
  - 和"垫子"的关联度 = 0.1
  - 和"开心"的关联度 = 0.05
```

自注意力让序列中的**每个位置都能直接关注任意其他位置**，不需要像 RNN 那样一步一步传递。

### 12.2 注意力的计算

每个位置生成三个向量：

- **Q（Query）**：我在找什么信息？
- **K（Key）**：我有什么信息可以提供？
- **V（Value）**：我实际要传递的内容是什么？

$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right) V$$

直觉：Q 和 K 做内积算"相关度"，相关度高的 V 被赋予更大的权重。

### 12.3 多头注意力（Multi-Head Attention）

把 Q、K、V 各拆成多份，每份独立做注意力，最后拼回来。

意义：不同的"头"可以关注不同类型的关系。比如一个头关注语法结构，另一个头关注语义相似性。

### 12.4 Transformer 的完整结构

```text
编码器（Encoder）×N：              解码器（Decoder）×N：
┌─────────────────┐              ┌─────────────────┐
│ 自注意力          │              │ 掩码自注意力      │
│ Add & LayerNorm  │              │ Add & LayerNorm  │
│ 前馈网络（FFN）   │              │ 交叉注意力        │
│ Add & LayerNorm  │              │ Add & LayerNorm  │
└─────────────────┘              │ 前馈网络（FFN）   │
                                 │ Add & LayerNorm  │
                                 └─────────────────┘
```

### 12.5 位置编码

注意力机制本身不区分位置（"猫咬狗"和"狗咬猫"在纯注意力看来是一样的）。位置编码给每个位置加上一个独特的信号，让模型知道"谁在前谁在后"。

### 12.6 Transformer 为什么这么强

| 优势 | 解释 |
|------|------|
| **全局视野** | 每个位置都能直接看到所有其他位置，不像 RNN 需要逐步传递 |
| **高度并行** | 所有位置可以同时计算，充分利用 GPU |
| **扩展性好** | 模型越大、数据越多、效果越好（Scaling Law） |
| **通用性强** | 同一个架构可以处理文本、图像、音频、甚至蛋白质序列 |

## 13. 其他重要架构

### 13.1 自编码器（AutoEncoder）

```text
输入 → 编码器 → 低维表示（瓶颈层）→ 解码器 → 重构输入
```

训练目标：让输出尽可能接近输入。瓶颈层被迫学会一个**压缩表示**。

用途：降维、去噪、特征学习。

### 13.2 变分自编码器（VAE）

在自编码器的基础上，瓶颈层不再是一个确定的向量，而是一个**概率分布**。从这个分布中采样，再送入解码器。

用途：生成新样本（比如生成新的人脸图片）。

### 13.3 GAN（生成对抗网络）

```text
生成器 G：噪声 → 假图片
判别器 D：图片 → 真/假

G 的目标：骗过 D（让 D 分不清真假）
D 的目标：识破 G（正确区分真假）
两者对抗训练，最终 G 能生成以假乱真的图片
```

用途：图像生成、超分辨率、风格迁移。

### 13.4 扩散模型（Diffusion Model）

```text
正向过程：真实图片 → 逐步加噪声 → 纯噪声
反向过程：纯噪声 → 逐步去噪声 → 生成图片（网络学的就是这一步）
```

当前最强的图像生成方法（Stable Diffusion、DALL-E、Midjourney 都基于此）。

### 13.5 图神经网络（GNN）

处理**图结构数据**（社交网络、分子结构、知识图谱）。每个节点聚合邻居的信息来更新自身表示。

# 第四部分 应用层：深度学习能做什么

## 14. 计算机视觉（CV）

| 任务 | 做什么 | 代表方法 |
|------|--------|---------|
| **图像分类** | 一张图 → 一个类别 | ResNet、EfficientNet、ViT |
| **目标检测** | 图中有什么、在哪里 | YOLO、Faster R-CNN、DETR |
| **语义分割** | 每个像素属于哪个类别 | U-Net、DeepLab、SegFormer |
| **实例分割** | 语义分割 + 区分不同个体 | Mask R-CNN、SAM |
| **图像生成** | 从无到有创造图片 | GAN、Diffusion Model |
| **超分辨率** | 低分辨率 → 高分辨率 | ESRGAN、SwinIR |
| **OCR** | 图片中的文字 → 字符串 | CRNN、PaddleOCR |

### Vision Transformer（ViT）

把图片切成 16×16 的小块（patch），每个 patch 当作一个"词"，送入标准 Transformer。证明了 Transformer 在视觉领域也能和 CNN 媲美甚至超越。

# 第五部分 工程层：怎么把模型训好

> 理论再好，训不出来也白搭。这部分是实战必备的"工程直觉"。

## 19. 数据

**数据决定了模型能力的上限，算法只是在逼近这个上限。**

| 主题 | 要点 |
|------|------|
| **数据清洗** | 去除错误标注、重复样本、异常值 |
| **数据增强** | 随机裁剪、翻转、颜色抖动、Mixup、CutMix |
| **数据划分** | 训练集 / 验证集 / 测试集必须分开，不能数据泄露 |
| **类别不均衡** | 过采样、欠采样、Focal Loss、加权采样 |

## 20. 训练技巧

| 技巧 | 解释 |
|------|------|
| **学习率 Warmup** | 开头几个 epoch 用小学习率，避免一开始就"跑偏" |
| **梯度裁剪** | 梯度太大时截断，防止梯度爆炸 |
| **混合精度训练（AMP）** | 部分计算用 float16，速度快一倍，显存省一半 |
| **梯度累积** | 显存不够时，多跑几个小 batch 再更新一次，等效于大 batch |
| **EMA（指数移动平均）** | 维护一份参数的平滑版本用于评估，通常效果更好 |

## 21. 调参指南

### 21.1 调参优先级

```text
最重要 ─────────────────────────────────→ 最不重要

学习率 > batch大小 > 模型深度/宽度 > 正则化强度 > 优化器类型
```

### 21.2 实用调参流程

1. **先跑通**：小数据、少 epoch、确认代码没 bug
2. **找学习率**：从 1e-3 开始试，看 loss 能不能下降
3. **调 batch size**：在显存允许的范围内尽量大
4. **加正则化**：如果训练集 acc 高但验证集 acc 低 → 过拟合了 → 加 Dropout / 数据增强
5. **加训练轮数**：如果验证集 acc 还在涨就继续训
6. **学习率调度**：加 Cosine Annealing 或 StepLR

## 22. 评估指标

| 指标 | 适用场景 | 含义 |
|------|---------|------|
| **准确率（Accuracy）** | 类别均衡的分类 | 预测对了多少比例 |
| **精确率（Precision）** | 关心"报出来的准不准" | 预测为正的里面有多少真的是正 |
| **召回率（Recall）** | 关心"漏了多少" | 真实为正的里面有多少被找到 |
| **F1** | 精确率和召回率的调和平均 | 综合指标 |
| **AUC-ROC** | 二分类 | 模型区分正负样本的能力 |
| **mAP** | 目标检测 | 多类别的平均精度 |
| **IoU** | 分割任务 | 预测区域和真实区域的重叠比 |
| **BLEU / ROUGE** | 文本生成 | 生成文本和参考文本的相似度 |
| **困惑度（Perplexity）** | 语言模型 | 模型对下一个词的"惊讶程度" |

## 23. 部署与推理优化

| 技术 | 作用 |
|------|------|
| **模型量化** | 把 float32 参数压缩到 int8，模型体积缩小 4 倍，推理加速 |
| **模型剪枝** | 去掉不重要的连接或通道，减少计算量 |
| **知识蒸馏** | 用大模型（教师）指导小模型（学生）学习 |
| **ONNX 导出** | 把 PyTorch 模型转成通用格式，方便跨平台部署 |
| **TensorRT** | NVIDIA 的推理加速引擎 |


# 附录：核心术语速查表

| 术语 | 一句话解释 |
|------|-----------|
| 张量（Tensor） | 多维数组，深度学习中数据和参数的基本载体 |
| 前向传播 | 输入 → 逐层计算 → 输出预测 |
| 反向传播 | 从损失出发，逆序计算每个参数的梯度 |
| 梯度下降 | 沿梯度反方向更新参数，让损失变小 |
| epoch | 完整遍历训练集一次 |
| batch | 一次送入模型的一小批样本 |
| 过拟合 | 训练集表现好、测试集表现差——模型"背答案"了 |
| 欠拟合 | 训练集表现都不好——模型太简单或没训够 |
| 超参数 | 需要人为设定的参数（学习率、层数、Dropout 率等） |
| 微调（Fine-tuning） | 拿预训练好的模型，在自己的数据上继续训练 |
| 迁移学习 | 把在大数据集上学到的知识，迁移到小数据集上 |
| 端到端（End-to-End） | 从原始输入直接到最终输出，中间不需要人工设计特征 |
| Embedding | 把离散符号（词、类别）映射成连续向量 |
| Attention | 让模型学会"该关注哪里" |
| Tokenization | 把文本拆成词或子词的过程 |
| Inference | 训练好的模型做预测的过程 |
| Latent Space | 隐空间，模型内部学到的压缩表示所在的空间 |
| Scaling Law | 模型越大、数据越多、计算越充分，效果就越好（对数增长） |
