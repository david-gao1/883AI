## 概念和代码，一张表对齐

下面这张表，方便你**从文章跳回屏幕**：左边是《什么是Q-learning》里常提的点，右边是**本作业里一眼能看到的写法**。

| 文章里讲到的点 | 在代码里怎么看 |
|----------------|----------------|
| 约 500×6，共 3000 格 | `q = np.zeros((n_states, n_actions))`，Taxi-v3 下形状是 `(500, 6)` |
| 行不是「第几步」，是现场编号 | `reset` / `step` 给的整数当 `q[state]` 的行下标 |
| 每步得分是 \(r\)，不是 \(Q\) | `reward` 来自 `env.step`；改的是 `q[state, action]` |
| TD 目标：\(r + \gamma\max Q(s')\)，结束只用 \(r\) | `target = reward` 或 `reward + gamma * np.max(q[next_state])` |
| 一开始目标常不准 | **没有单独判断**；同一公式一直算，靠多局训练把整张 `q` 磨准 |
| 「挪一小步」= 学习率，不是六种动作里选一个 | `q[state, action] += alpha * (target - q[state, action])` |
| 训练要探索，评估要贪心 | `epsilon_greedy` 训练；`evaluate_greedy` 里每步 `argmax` |
| 离策略：手可以乱试，算账用 max | **选动作**：ε 随机；**算 target**：`np.max(q[next_state])` |

更细的叙述见《什么是Q-learning》**第 2～5 节**（表长啥样、\(Q\) 与 \(r\)、怎么更新、怎么探索）。下面按 **`q_learning_taxi.py` 的执行顺序**走读。

## 2. Q 表：在哪创建、下标怎么理解？

```64:66:研究生作业/强化学习/q_learning_taxi.py
    n_states = env.observation_space.n
    n_actions = env.action_space.n
    q = np.zeros((n_states, n_actions), dtype=np.float64)
```

- `n_states`、`n_actions` 由环境决定；Taxi-v3 是 **500** 和 **6**。  
- **`q[state]`**：一行 6 个数，表示**当前现场**下六个动作的当前估计。  
- **`q[state, action]`**：本步要更新的那一格。

## 3. 训练：一整段主循环在干什么？

一句话顺序：**选动作 → `step` → 算 `target` → 用 `alpha` 改表 → `state` 换成下一现场**。

```69:90:研究生作业/强化学习/q_learning_taxi.py
    epsilon = epsilon_start
    for ep in range(n_episodes):
        state, _ = env.reset(seed=seed + ep)
        done = False
        ep_return = 0.0

        while not done:
            action = epsilon_greedy(q[state], n_actions, epsilon, rng)
            next_state, reward, terminated, truncated, _ = env.step(action)
            done = terminated or truncated

            if done:
                target = reward
            else:
                target = reward + gamma * float(np.max(q[next_state]))

            q[state, action] += alpha * (target - q[state, action])
            ep_return += reward
            state = next_state

        episode_returns.append(ep_return)
        epsilon = max(epsilon_end, epsilon * epsilon_decay)
```

**建议对照文章第 4 节，重点看三处代码：**

**（1）`target = reward + gamma * np.max(q[next_state])`（还没结束时）**  
这就是 **TD 目标**。训练初期 **`q` 还很糙**，这里的 `target` 也会偏——**正常**，不用写 `if 早期` 特判，靠后面成千上万步一起把表练好。

**（2）`if done: target = reward`**  
本局结束（`terminated` 或 `truncated`）时，**不再**往后面猜价值，目标里**只有**这一步的 `reward`。对应文章里「终局不做 bootstrap」。

**（3）`q[state, action] += alpha * (target - q[state, action])`**  
即 **\(Q \leftarrow Q + \alpha(\text{目标} - Q)\)**。**`alpha`** 就是「挪一小步」。若写成 `q[state, action] = target`，相当于 **\(\alpha=1\)**，往往更容易抖。

**补充两行：**  
- **`ep_return`**：把本局每一步的 `reward` 加起来，只给**画图、打印**用，**不参与**上面的更新公式。  
- **每局末尾的 `epsilon = max(...)`**：探索率慢慢降下来，对应文章第 5 节「前面多试、后面少试」。

## 4. 探索：`epsilon_greedy`

```45:50:研究生作业/强化学习/q_learning_taxi.py
def epsilon_greedy(
    q_row: np.ndarray, n_actions: int, epsilon: float, rng: random.Random
) -> int:
    if rng.random() < epsilon:
        return rng.randrange(n_actions)
    return int(np.argmax(q_row))
```

- 传入的 **`q_row`** 就是 **`q[state]`** 这一行。  
- **`argmax`**：贪心，选当前估计最好的列。  
- **`randrange`**：以小概率均匀随机选一列，避免永远试不全。

更新时用的 **`np.max(q[next_state])`** 是在假设「到了下一现场会按**最优列**走」，和「下一步你真随机还是贪心」可以不一致——这就是实现上的**离策略**组合。

## 5. 评估：`evaluate_greedy`（不用 ε）

```105:107:研究生作业/强化学习/q_learning_taxi.py
        while not done:
            action = int(np.argmax(q[state]))
            state, reward, terminated, truncated, _ = env.step(action)
```

训练要带随机；**看学得怎么样**时，通常用**纯贪心**：每步只看 **`argmax`**，跑多局算平均回报、平均步数，和训练曲线一起看。

## 6. `main`：先训练，再评估，再可选出图

```188:218:研究生作业/强化学习/q_learning_taxi.py
    train_env = gym.make("Taxi-v3")
    q, returns = q_learning_train(
```

## 7. 曲线图在表达什么？

- **`moving_average`**：给每局总回报做滑动平均，曲线不那么毛刺。  
- **`save_result_plots`**：浅蓝线 = 每局回报，橙线 = 滑动平均；角上文字是贪心评估摘要。  

Taxi 每步常是 **−1**，回报多在**负区间**；曲线**整体往上走**，一般说明越学越会完成任务（与《什么是Q-learning》第 3 节对照）。

## 9. 超参数和变量对应

| 命令行 / 含义 | 代码里 | 一句话 |
|---------------|--------|--------|
| `--alpha` | `alpha` | 每次把表改多少，即「小步」多大 |
| `--gamma` | `gamma` | 进 `target`，越远越打折 |
