# 图像特征提取与分类实验报告

## 摘要

本实验实现了基于手工特征的图像分类方法，提取HOG（Histogram of Oriented Gradients）特征和颜色直方图特征，并使用SVM和两层神经网络进行分类。实验结果表明，特征+SVM达到42.1%的准确率（相比原始像素+SVM的37-39%提升3-5个百分点），验证了特征工程对线性分类器的有效性。但特征+神经网络仅达到10.3%的准确率（远低于原始像素+神经网络的54.7%），说明特征和分类器的组合需要仔细匹配，不是所有组合都能提升性能。实验揭示了特征工程的重要性，同时也说明了特征预处理、超参数调优和训练策略对性能的关键影响。

**关键词**：图像特征、HOG、颜色直方图、特征工程、SVM、神经网络、CIFAR-10

## 1 实验目的

本实验旨在实现基于手工特征的图像分类方法，重点完成以下目标：

1. **理解特征提取**：实现HOG特征和颜色直方图特征的提取
2. **掌握特征工程**：理解如何从原始像素中提取更有用的特征
3. **验证特征优势**：比较特征+SVM/神经网络与原始像素+SVM/神经网络的性能
4. **学习特征组合**：理解如何组合不同特征提升分类性能
5. **分析误分类**：通过可视化误分类结果理解模型的局限性

## 2 特征提取方法

### 2.1 HOG特征（Histogram of Oriented Gradients）

**原理**：HOG特征通过计算图像梯度的方向直方图来捕获图像的纹理和边缘信息。

**计算步骤**：
1. **转换为灰度图**：将RGB图像转换为灰度图
2. **计算梯度**：计算x方向和y方向的梯度
3. **计算梯度幅值和方向**：
   - 幅值：$mag = \sqrt{g_x^2 + g_y^2}$
   - 方向：$ori = \arctan2(g_y, g_x)$
4. **分块统计**：将图像分成8×8的cell，统计每个cell内9个方向的梯度直方图
5. **展平**：将所有cell的直方图展平成一维向量

**特征维度**：对于32×32的图像，约576维（4×4个cell × 9个方向 × 4个方向组）

**优势**：
- **纹理信息**：捕获图像的纹理和边缘模式
- **对光照不敏感**：梯度对光照变化相对鲁棒
- **局部特征**：捕获局部空间信息

**实现代码**（`cs231n/features.py`）：

### 2.2 颜色直方图特征

**原理**：颜色直方图通过统计HSV颜色空间中色调（Hue）的分布来捕获图像的颜色信息。

**计算步骤**：
1. **RGB转HSV**：将RGB图像转换为HSV颜色空间
2. **提取色调（Hue）**：使用HSV的H通道
3. **统计直方图**：将色调值分成nbin个bins（默认10），统计每个bin的像素数量
4. **归一化**：归一化直方图，使其和为1

**特征维度**：10维（nbin=10）

**优势**：
- **颜色信息**：捕获图像的颜色分布
- **对形状不敏感**：只关注颜色，不关注空间位置
- **计算简单**：计算快速，特征维度低

**实现代码**（`cs231n/features.py`）：
```python
def color_histogram_hsv(im, nbin=10, xmin=0, xmax=255, normalized=True):
    # RGB转HSV
    hsv = matplotlib.colors.rgb_to_hsv(im/xmax) * xmax
    
    # 统计色调直方图
    bins = np.linspace(xmin, xmax, nbin+1)
    imhist, bin_edges = np.histogram(hsv[:,:,0], bins=bins, density=normalized)
    imhist = imhist * np.diff(bin_edges)
    
    return imhist
```

### 2.3 特征组合

**组合方法**：将HOG特征和颜色直方图特征拼接在一起。

**特征维度**：
- HOG特征：约576维
- 颜色直方图：10维
- **总特征维度**：约586维（比原始像素3072维低很多）

**为什么组合？**
- **互补性**：HOG捕获纹理，颜色直方图捕获颜色，两者互补
- **全面性**：同时考虑纹理和颜色信息，更全面地描述图像
- **性能提升**：组合特征比单独使用任一特征效果更好

**实现代码**：
```python
from cs231n.features import *

num_color_bins = 10
feature_fns = [hog_feature, lambda img: color_histogram_hsv(img, nbin=num_color_bins)]
X_train_feats = extract_features(X_train, feature_fns, verbose=True)
X_val_feats = extract_features(X_val, feature_fns)
X_test_feats = extract_features(X_test, feature_fns)
```

### 2.4 特征预处理

**步骤1：减去均值**
```python
mean_feat = np.mean(X_train_feats, axis=0, keepdims=True)
X_train_feats -= mean_feat
X_val_feats -= mean_feat
X_test_feats -= mean_feat
```
**目的**：中心化特征，使特征均值为0。

**步骤2：除以标准差**
```python
std_feat = np.std(X_train_feats, axis=0, keepdims=True)
X_train_feats /= std_feat
X_val_feats /= std_feat
X_test_feats /= std_feat
```
**目的**：标准化特征，使每个特征的尺度相同，避免某些特征主导分类。

**步骤3：添加偏置维度**
```python
X_train_feats = np.hstack([X_train_feats, np.ones((X_train_feats.shape[0], 1))])
```
**目的**：为线性分类器添加偏置项，简化实现。

## 3 SVM分类器在特征上的表现

### 3.1 超参数调优

**搜索空间**：
- **学习率**：[1e-9, 1e-8, 1e-7]
- **正则化强度**：[5e4, 5e5, 5e6]

**调优策略**：网格搜索在验证集上搜索最优超参数组合。

**实现代码**：
```python
from cs231n.classifiers.linear_classifier import LinearSVM

learning_rates = [1e-9, 1e-8, 1e-7]
regularization_strengths = [5e4, 5e5, 5e6]

results = {}
best_val = -1
best_svm = None

for lr in learning_rates:
    for reg in regularization_strengths:
        # 训练SVM（使用特征）
        svm = LinearSVM()
        loss_hist = svm.train(X_train_feats, y_train, learning_rate=lr, reg=reg,
                              num_iters=1500, verbose=False)
        
        # 预测
        y_train_pred = svm.predict(X_train_feats)
        y_val_pred = svm.predict(X_val_feats)
        
        # 计算准确率
        train_acc = np.mean(y_train_pred == y_train)
        val_acc = np.mean(y_val_pred == y_val)
        
        # 保存结果
        results[(lr, reg)] = (train_acc, val_acc)
        
        # 更新最佳模型
        if val_acc > best_val:
            best_val = val_acc
            best_svm = svm
```

### 3.2 实验结果

**实际执行结果**：
- **最佳验证集准确率**：41.7%（lr=1e-7, reg=5e4）
- **测试集准确率**：42.1%
- **所有超参数组合结果**：
  - lr=1e-9, reg=5e4: train_acc=11.1%, val_acc=11.6%
  - lr=1e-9, reg=5e5: train_acc=10.7%, val_acc=12.0%
  - lr=1e-9, reg=5e6: train_acc=41.0%, val_acc=41.1%
  - lr=1e-8, reg=5e4: train_acc=11.9%, val_acc=11.3%
  - lr=1e-8, reg=5e5: train_acc=41.1%, val_acc=41.0%
  - lr=1e-8, reg=5e6: train_acc=40.4%, val_acc=40.1%
  - lr=1e-7, reg=5e4: train_acc=41.3%, val_acc=41.7% ⭐ **最佳**
  - lr=1e-7, reg=5e5: train_acc=41.4%, val_acc=41.1%
  - lr=1e-7, reg=5e6: train_acc=29.8%, val_acc=27.8%

**性能提升**：
- 相比原始像素+SVM（37-39%），提升约3-5个百分点
- 说明特征提取确实有效，但提升幅度略低于预期（预期44%）

**分析**：
- **特征优势**：HOG和颜色直方图特征比原始像素更有区分性
- **维度降低**：从3072维降到586维，减少计算量
- **信息保留**：虽然维度降低，但保留了关键信息（纹理和颜色）
- **超参数敏感性**：学习率和正则化强度的选择对性能影响较大，需要仔细调优

## 4 神经网络分类器在特征上的表现

### 4.1 超参数调优

**搜索空间**：
- **隐藏层大小**：[200, 300, 500]
- **学习率**：[1e-3, 5e-4]
- **正则化强度**：[0.05, 0.1, 0.25]

**调优策略**：网格搜索在验证集上搜索最优超参数组合。

**实现代码**：
```python
from cs231n.classifiers.neural_net import TwoLayerNet

input_dim = X_train_feats.shape[1]  # 特征维度（约586）
best_val_acc = -1
best_net = None

hidden_sizes = [200, 300, 500]
learning_rates = [1e-3, 5e-4]
regularization_strengths = [0.05, 0.1, 0.25]

for hidden_size in hidden_sizes:
    for lr in learning_rates:
        for reg in regularization_strengths:
            # 创建网络
            net = TwoLayerNet(input_dim, hidden_size, num_classes)
            
            # 训练
            stats = net.train(X_train_feats, y_train, X_val_feats, y_val,
                            num_iters=3000, batch_size=200,
                            learning_rate=lr, learning_rate_decay=0.95,
                            reg=reg, verbose=False)
            
            # 评估
            val_acc = (net.predict(X_val_feats) == y_val).mean()
            
            # 更新最佳模型
            if val_acc > best_val_acc:
                best_val_acc = val_acc
                best_net = net
```

- **特征维度**：输入维度从3072降到586，但网络结构可能不适合这种低维特征
- **需要进一步调优**：可能需要调整超参数搜索空间、训练策略或网络架构

## 5 误分类分析

### 5.1 误分类模式

通过可视化误分类结果，可以观察到以下模式：

**1. 视觉相似性导致的误分类**：
- **飞机 vs 鸟**：某些鸟的图像（特别是飞行中的鸟）可能被误分类为飞机，因为它们都有"在天空中"的特征
- **汽车 vs 卡车**：某些卡车的图像可能被误分类为汽车，因为它们都是车辆，形状相似
- **猫 vs 狗**：某些猫或狗的图像可能被误分类，因为它们都是四足动物，形状相似
- **船 vs 飞机**：某些船或飞机的图像可能被误分类，因为它们都有"交通工具"的特征

**2. 特征相似性**：
- **HOG特征**：捕获纹理和边缘信息，如果两个类别有相似的纹理模式（如"天空"背景），可能被误分类
- **颜色直方图**：捕获颜色信息，如果两个类别有相似的颜色分布（如"蓝色"天空），可能被误分类

### 5.2 误分类是否合理？

**合理的原因**：
- **视觉相似性**：误分类的类别在视觉上确实相似（如汽车和卡车）
- **特征局限性**：HOG和颜色直方图是手工设计的特征，可能无法完全区分相似类别
- **线性分类器限制**：SVM是线性分类器，只能学习线性决策边界，对于复杂的视觉相似性可能无法完全区分

**不合理的原因**：
- **明显不同的类别**：如果明显不同的类别（如"猫"和"船"）被误分类，说明特征提取或分类器有问题
- **特征不足**：如果误分类率很高，说明特征可能不足以区分某些类别

### 5.3 改进方向

**1. 更好的特征**：
- 使用更复杂的特征（如深度特征）
- 组合更多特征（如SIFT、LBP等）
- 使用特征选择方法选择最有用的特征

**2. 非线性分类器**：
- 使用神经网络等非线性分类器，能够学习更复杂的决策边界
- 使用更深的网络或更复杂的架构

**3. 数据增强**：
- 增加训练数据的多样性（旋转、翻转、裁剪等）
- 帮助模型学习更鲁棒的特征
