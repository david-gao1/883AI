# 两层神经网络：非线性激活与反向传播的双重策略

> 本文聚焦两层全连接神经网络的核心设计思路和算法原理，解释代码背后的设计原因和数学基础。

---

## 📖 核心概念

**两层全连接神经网络**：非线性分类器，通过多层变换实现分类。架构：输入 → 全连接层1 → ReLU → 全连接层2 → Softmax损失。

**ReLU激活函数**：$\text{ReLU}(x) = \max(0, x)$，引入非线性，使模型能够学习非线性决策边界。

**公式解释**：
- **$\max(0, x)$**：逐元素取最大值
  - 当$x > 0$时，$\text{ReLU}(x) = x$
  - 当$x \leq 0$时，$\text{ReLU}(x) = 0$
- **梯度**：$\frac{\partial \text{ReLU}(x)}{\partial x} = \begin{cases} 1 & \text{if } x > 0 \\ 0 & \text{if } x \leq 0 \end{cases}$
- **优势**：计算简单，梯度稳定（正区域梯度为1），避免梯度消失

**反向传播（Backpropagation）**：使用链式法则计算梯度，从输出层向输入层逐层传播误差。

**随机梯度下降（SGD）**：每次迭代随机采样batch计算梯度，比全批量梯度下降快，适合大规模数据。

**正则化**：$R(W) = \lambda (\sum W_{1,ij}^2 + \sum W_{2,ij}^2)$，防止过拟合，限制模型复杂度。

**公式解释**：
- **$W_{1,ij}$**：第一层权重矩阵的第$(i,j)$个元素
- **$W_{2,ij}$**：第二层权重矩阵的第$(i,j)$个元素
- **$\sum W_{1,ij}^2$**：第一层所有权重的平方和（L2范数的平方）
- **$\sum W_{2,ij}^2$**：第二层所有权重的平方和
- **$\lambda$**：正则化强度（超参数），控制正则化项的权重
- **作用**：惩罚权重大大，防止模型过于复杂，提高泛化能力
- **梯度**：$\frac{\partial R(W)}{\partial W_1} = 2\lambda W_1$，$\frac{\partial R(W)}{\partial W_2} = 2\lambda W_2$

## 📚 核心逻辑：用非线性激活与反向传播实现复杂分类

图像分类需要学习分类规则。线性分类器（SVM 37-39%，Softmax 33.8%）只能学习线性决策边界，准确率有限。**神经网络通过ReLU激活引入非线性，能够学习非线性决策边界，准确率显著提升（48-52%）**。这是本文的核心结论。

**为什么需要神经网络？** 线性分类器的局限在于只能学习线性决策边界。对于复杂的图像分类任务，数据往往不是线性可分的，需要非线性模型。神经网络通过ReLU激活函数引入非线性变换，使模型能够学习更复杂的特征组合和决策边界。

**但实现神经网络面临两个关键问题**：

**问题1：梯度计算复杂**。神经网络有多层参数（$W_1, b_1, W_2, b_2$），需要计算每层参数的梯度。如果手动推导每层的梯度公式，计算量大且容易出错。如何高效准确地计算所有参数的梯度？

**问题2：超参数难选**。神经网络有更多超参数（隐藏层大小、学习率、正则化强度），每个超参数的选择都影响模型性能。如何选择最优的超参数组合？

**解决方案：采用反向传播算法和验证集调优的双重策略**。

**解决方案1：反向传播算法**。通过链式法则从输出层向输入层逐层计算梯度，一次前向传播和一次反向传播就能计算所有参数的梯度，高效且准确。这解决了梯度计算复杂的问题。

**解决方案2：验证集调优**。通过网格搜索在验证集上搜索最优超参数组合（隐藏层大小、学习率、正则化强度），避免过拟合测试集，确保选择的超参数能够泛化到新数据。这解决了超参数难选的问题。

**但仅有这两个解决方案还不够**。为了进一步提升分类效果，神经网络还需要两个核心机制：

**核心机制1：ReLU激活引入非线性**。ReLU激活通过$\max(0, x)$引入非线性，使模型能够学习非线性决策边界。这是神经网络相比线性分类器的关键优势，也是准确率提升的根本原因。

**核心机制2：SGD高效优化**。SGD通过随机采样batch计算梯度，比全批量梯度下降快，适合大规模数据。这使得训练过程高效，能够快速迭代和调优。

**四个机制如何协同工作？** 反向传播高效计算梯度，验证集调优选择最优超参数，ReLU激活引入非线性，SGD高效优化。这四个机制协同工作，共同实现了高效准确的分类。

**最终效果如何？** 在CIFAR-10数据集上，默认参数（hidden_size=50, learning_rate=1e-4, reg=0.25, num_iters=1000）下验证集准确率为28.7%，说明模型容量不足。通过超参数调优（网格搜索36种组合，最优组合：hidden_size=300, learning_rate=1e-3, reg=0.05, num_iters=3000），验证集准确率达到55%，测试集准确率达到54.7%，虽然不如深度学习方法（通常>90%），但相比线性分类器（SVM 37-39%，Softmax 33.8%）有显著提升（+15-20个百分点），证明了非线性模型的优势。更重要的是，测试集准确率与验证集准确率接近（54.7% vs 55%），说明模型泛化良好，没有过拟合。

**核心逻辑链条**：任务（图像分类）→ 方法选择（神经网络非线性模型）→ 问题转化（多层变换优化）→ 关键问题（梯度计算复杂+超参数难选）→ 解决方案（反向传播+验证集调优）→ 核心机制（高效梯度计算+最优选择+非线性激活+优化方法）→ 最终效果（快速训练+较高准确率）

## 一、算法设计：前向传播与反向传播

基于核心逻辑，算法设计需要实现前向传播（计算得分）和反向传播（计算梯度）。下面分别说明设计思路。

### 1.1 前向传播设计

**目标**：计算所有样本的类别得分。

**数学公式**：

**第一层全连接**：
$$z_1 = X \cdot W_1 + b_1$$

**第一层要做什么？**

**目的**：将原始输入特征（3072维图像像素）转换为隐藏层特征（H维，如300维），学习输入特征的有用组合。

**为什么需要第一层？**
- **原始输入**：图像展平后是3072个像素值，这些是"原始特征"，直接用于分类效果不好
- **第一层的作用**：学习如何组合这3072个像素，提取更有用的特征
- **类比**：就像从"像素"提取"边缘"、"纹理"等更高级的特征

**具体例子说明**：

假设我们有一个简单的例子：
- **输入**：$X$形状$(2, 4)$，表示2个样本，每个样本4个特征
- **权重**：$W_1$形状$(4, 3)$，表示4个输入特征映射到3个隐藏特征
- **偏置**：$b_1$形状$(3,)$，3个隐藏单元各有一个偏置

**计算过程**：

```python
# 输入：2个样本，每个样本4个特征
X = [[x1, x2, x3, x4],    # 样本1的4个特征
     [x5, x6, x7, x8]]    # 样本2的4个特征
# 形状：(2, 4)

# 权重：4个输入特征 → 3个隐藏特征
W1 = [[w11, w12, w13],    # 输入特征1到3个隐藏单元的权重
      [w21, w22, w23],    # 输入特征2到3个隐藏单元的权重
      [w31, w32, w33],    # 输入特征3到3个隐藏单元的权重
      [w41, w42, w43]]    # 输入特征4到3个隐藏单元的权重
# 形状：(4, 3)

# 偏置：3个隐藏单元各有一个偏置
b1 = [b1, b2, b3]
# 形状：(3,)

# 计算：z1 = X · W1 + b1
# 样本1的第1个隐藏特征：
z1[0, 0] = x1*w11 + x2*w21 + x3*w31 + x4*w41 + b1
# 样本1的第2个隐藏特征：
z1[0, 1] = x1*w12 + x2*w22 + x3*w32 + x4*w42 + b2
# 样本1的第3个隐藏特征：
z1[0, 2] = x1*w13 + x2*w23 + x3*w33 + x4*w43 + b3

# 样本2的计算类似
```

**矩阵乘法详解**：

$$z_1[i, j] = \sum_{k=1}^{D} X[i, k] \cdot W_1[k, j] + b_1[j]$$

**公式解释**：
- **$z_1[i, j]$**：样本$i$的第$j$个隐藏特征
- **$X[i, k]$**：样本$i$的第$k$个输入特征（像素值）
- **$W_1[k, j]$**：输入特征$k$到隐藏特征$j$的权重
- **$\sum_{k=1}^{D}$**：对所有输入特征求和（加权求和）
- **$b_1[j]$**：隐藏特征$j$的偏置

**实际例子（图像分类）**：

假设输入是一张32×32×3的图像：
- **展平后**：$X[i]$是3072维向量（32×32×3=3072个像素值）
- **第一层权重**：$W_1$形状$(3072, 300)$，表示3072个像素映射到300个隐藏特征
- **计算过程**：
  - 隐藏特征1：$z_1[i, 1] = X[i, 1] \cdot W_1[1, 1] + X[i, 2] \cdot W_1[2, 1] + ... + X[i, 3072] \cdot W_1[3072, 1] + b_1[1]$
  - 这是3072个像素的加权求和，权重$W_1[:, 1]$可能学习到"检测某个边缘模式"
  - 隐藏特征2：$z_1[i, 2]$可能学习到"检测另一个模式"（如纹理、颜色等）
  - ...共300个隐藏特征，每个学习不同的特征模式

**维度变化**：
- **输入**：$(N, 3072)$ → **输出**：$(N, 300)$
- **维度降低**：从3072维降到300维（如果$H=300$）
- **特征变换**：从"原始像素"变为"学习到的特征组合"

**公式解释**：
- **$X$**：输入数据矩阵，形状$(N, D)$，其中$N$是batch大小（样本数），$D=3072$是输入维度（32×32×3图像展平）
- **$W_1$**：第一层权重矩阵，形状$(D, H)$，其中$H$是隐藏层大小（如300）
  - 每一列$W_1[:, j]$表示如何组合输入特征得到第$j$个隐藏特征
  - 每一行$W_1[k, :]$表示输入特征$k$对各个隐藏特征的贡献
- **$b_1$**：第一层偏置向量，形状$(H,)$，广播到$(N, H)$
  - 偏置允许隐藏特征在没有输入时也有非零值
- **$z_1$**：第一层线性输出，形状$(N, H)$，表示每个样本经过线性变换后的特征
  - $z_1[i, j]$是样本$i$的第$j$个隐藏特征的"原始得分"（未经过激活函数）
- **计算过程**：对每个样本$X[i]$，计算$X[i] \cdot W_1 + b_1$，得到$z_1[i]$，共$N$个样本
  - 矩阵乘法$X \cdot W_1$一次性计算所有样本的所有隐藏特征
  - 偏置$b_1$通过广播加到每一行

**ReLU激活**：
$$a_1 = \text{ReLU}(z_1) = \max(0, z_1)$$

**ReLU激活要做什么？**

**目的**：对第一层的线性输出$z_1$进行非线性变换，引入非线性，使模型能够学习非线性决策边界。

**为什么需要ReLU？**

**问题**：如果没有ReLU，只有线性变换（$z_1 = X \cdot W_1 + b_1$，$scores = z_1 \cdot W_2 + b_2$），那么整个网络仍然是线性的：
$$scores = (X \cdot W_1 + b_1) \cdot W_2 + b_2 = X \cdot (W_1 \cdot W_2) + (b_1 \cdot W_2 + b_2) = X \cdot W_{combined} + b_{combined}$$

这等价于单层线性分类器，无法学习非线性决策边界。

**ReLU的作用**：
- **引入非线性**：$\max(0, z_1)$是非线性函数，使网络能够学习非线性决策边界
- **激活特性**：只保留正值，将负值置为0，相当于"激活"某些特征，"抑制"其他特征
- **稀疏性**：负值被置为0，产生稀疏表示，有助于特征选择

**具体例子说明**：

假设第一层输出$z_1$为：
```python
# 样本1的3个隐藏特征
z1[0] = [2.5, -1.3, 0.8]  # 第1个样本的3个隐藏特征

# 经过ReLU激活
a1[0] = [2.5, 0.0, 0.8]   # 保留正值，负值置为0
```

**计算过程**：
- $a_1[0, 0] = \max(0, 2.5) = 2.5$（正值保留）
- $a_1[0, 1] = \max(0, -1.3) = 0$（负值置为0）
- $a_1[0, 2] = \max(0, 0.8) = 0.8$（正值保留）

**实际意义**：
- **隐藏特征1**（值为2.5）：表示"检测到某个模式"（如边缘），激活强度2.5
- **隐藏特征2**（值为-1.3 → 0）：表示"未检测到某个模式"，被抑制
- **隐藏特征3**（值为0.8）：表示"检测到另一个模式"（如纹理），激活强度0.8

**为什么ReLU能学习非线性？**

**线性组合**：如果只有线性变换，无论多少层，最终都是输入的线性组合。

**非线性变换**：ReLU的$\max(0, x)$是非线性函数，它：
- 在正区域：$f(x) = x$（线性）
- 在负区域：$f(x) = 0$（常数）
- 整体：分段线性，但整体是非线性的

**多层组合**：多个ReLU层组合后，可以学习复杂的非线性决策边界。

**公式解释**：
- **$\max(0, z_1)$**：逐元素取最大值，$a_1[i,j] = \max(0, z_1[i,j])$
- **作用**：引入非线性，当$z_1[i,j] > 0$时，$a_1[i,j] = z_1[i,j]$（保留）；当$z_1[i,j] \leq 0$时，$a_1[i,j] = 0$（抑制）
- **形状**：$a_1$形状$(N, H)$，与$z_1$相同
- **意义**：这是神经网络的关键优势，使模型能够学习非线性决策边界

**第二层全连接**：
$$scores = a_1 \cdot W_2 + b_2$$

**第二层要做什么？**

**目的**：将隐藏层特征（H维，如300维）映射到类别得分（C维，10个类别），用于最终分类。

**为什么需要第二层？**

**输入**：第一层+ReLU后得到$a_1$，形状$(N, H)$，表示每个样本的H个隐藏特征。

**任务**：需要将这H个隐藏特征转换为10个类别的得分，用于分类。

**第二层的作用**：
- **特征到类别的映射**：学习如何组合H个隐藏特征，得到每个类别的得分
- **分类决策**：得分最高的类别就是预测类别

**具体例子说明**：

假设隐藏层大小$H=3$，类别数$C=2$（简化例子）：
- **输入**：$a_1[i] = [2.5, 0.0, 0.8]$（样本$i$的3个隐藏特征）
- **权重**：$W_2$形状$(3, 2)$，表示3个隐藏特征到2个类别的映射
- **偏置**：$b_2 = [0.1, -0.2]$（2个类别各有一个偏置）

**计算过程**：

```python
# 权重矩阵：3个隐藏特征 → 2个类别
W2 = [[w11, w12],    # 隐藏特征1到2个类别的权重
      [w21, w22],    # 隐藏特征2到2个类别的权重
      [w31, w32]]    # 隐藏特征3到2个类别的权重
# 形状：(3, 2)

# 偏置：2个类别各有一个偏置
b2 = [b1, b2]
# 形状：(2,)

# 计算类别得分
# 类别1的得分：
scores[i, 0] = a1[i, 0]*W2[0, 0] + a1[i, 1]*W2[1, 0] + a1[i, 2]*W2[2, 0] + b2[0]
            = 2.5*w11 + 0.0*w21 + 0.8*w31 + b1
            = 2.5*w11 + 0.8*w31 + b1

# 类别2的得分：
scores[i, 1] = a1[i, 0]*W2[0, 1] + a1[i, 1]*W2[1, 1] + a1[i, 2]*W2[2, 1] + b2[1]
            = 2.5*w12 + 0.0*w22 + 0.8*w32 + b2
            = 2.5*w12 + 0.8*w32 + b2
```

**实际例子（图像分类）**：

假设隐藏层大小$H=300$，类别数$C=10$（CIFAR-10）：
- **输入**：$a_1[i]$是300维向量，表示样本$i$的300个隐藏特征
- **权重**：$W_2$形状$(300, 10)$，表示300个隐藏特征到10个类别的映射
- **计算过程**：
  - **类别0（飞机）的得分**：$scores[i, 0] = a_1[i, 1] \cdot W_2[1, 0] + a_1[i, 2] \cdot W_2[2, 0] + ... + a_1[i, 300] \cdot W_2[300, 0] + b_2[0]$
    - 这是300个隐藏特征的加权求和
    - 权重$W_2[:, 0]$学习到"哪些隐藏特征对识别飞机重要"
    - 例如：如果隐藏特征50（可能检测"天空"）和隐藏特征120（可能检测"机翼"）对飞机重要，则$W_2[50, 0]$和$W_2[120, 0]$会较大
  - **类别1（汽车）的得分**：$scores[i, 1]$类似，但权重$W_2[:, 1]$学习到"哪些隐藏特征对识别汽车重要"
  - ...共10个类别，每个类别都有自己的权重组合

**维度变化**：
- **输入**：$(N, 300)$ → **输出**：$(N, 10)$
- **维度降低**：从300维降到10维
- **特征变换**：从"隐藏特征"变为"类别得分"

**预测过程**：
```python
# 计算所有类别的得分
scores[i] = [scores[i, 0], scores[i, 1], ..., scores[i, 9]]
          = [飞机得分, 汽车得分, ..., 卡车得分]

# 选择得分最高的类别
predicted_class = argmax(scores[i])
# 例如：如果scores[i, 2]最大，则预测为类别2（鸟）
```

**公式解释**：
- **$W_2$**：第二层权重矩阵，形状$(H, C)$，其中$C=10$是类别数
  - 每一列$W_2[:, j]$表示如何组合隐藏特征得到类别$j$的得分
  - 每一行$W_2[k, :]$表示隐藏特征$k$对各个类别的贡献
- **$b_2$**：第二层偏置向量，形状$(C,)$，广播到$(N, C)$
  - 偏置允许类别在没有激活的隐藏特征时也有非零得分
- **$scores$**：类别得分矩阵，形状$(N, C)$，$scores[i,j]$表示样本$i$在类别$j$上的得分
  - 得分越高，表示样本越可能属于该类别
- **计算过程**：对每个样本$i$，计算$scores[i] = a_1[i] \cdot W_2 + b_2$，得到该样本在所有类别上的得分
  - 矩阵乘法$a_1 \cdot W_2$一次性计算所有样本的所有类别得分
  - 偏置$b_2$通过广播加到每一行

**设计原因**：全连接层学习特征组合，ReLU激活引入非线性（这是神经网络的关键优势），第二层将隐藏层特征映射到类别得分。

**实现代码**：
```python
# First layer: fully connected + ReLU
z1 = X.dot(W1) + b1  # (N, H)
a1 = np.maximum(0, z1)  # ReLU: (N, H)

# Second layer: fully connected
scores = a1.dot(W2) + b2  # (N, C)
```

**关键点**：前向传播时保存中间结果（$z_1, a_1$），供反向传播复用，提高效率。

### 1.2 损失函数设计

**目标**：衡量预测与真实分布的差异，并防止过拟合。

**数学公式**：

**Softmax概率**：
$$p_{y_i} = \frac{e^{s_{y_i}}}{\sum_{j=0}^{C-1} e^{s_j}}$$

**公式解释**：
- **$s_j$**：样本$i$在类别$j$上的得分，$scores[i,j]$
- **$s_{y_i}$**：样本$i$在正确类别$y_i$上的得分
- **$e^{s_j}$**：对得分取指数，将得分转换为正数
- **$\sum_{j=0}^{C-1} e^{s_j}$**：所有类别得分的指数和，用于归一化
- **$p_{y_i}$**：正确类别的softmax概率，范围$(0, 1)$
- **数值稳定性**：实际计算时，先计算$scores - \max(scores, axis=1)$，再取指数，避免溢出
- **意义**：将得分转换为概率分布，所有类别概率和为1

**交叉熵损失（数据损失）**：
$$L_{data} = \frac{1}{N}\sum_{i=1}^{N} \left(-\log(p_{y_i})\right)$$

**公式解释**：
- **$-\log(p_{y_i})$**：正确类别的负对数概率
- **为什么用负对数**：
  - 当$p_{y_i} = 1$（完全正确）时，$-\log(1) = 0$，损失为0
  - 当$p_{y_i} \to 0$（完全错误）时，$-\log(p_{y_i}) \to +\infty$，损失很大
  - 鼓励模型提高正确类别的概率
- **$\frac{1}{N}\sum_{i=1}^{N}$**：对所有样本求平均，得到平均损失
- **$L_{data}$**：数据损失，衡量预测与真实分布的差异

**L2正则化项**：
$$R(W) = \lambda \left(\sum_{i,j} W_{1,ij}^2 + \sum_{i,j} W_{2,ij}^2\right)$$

**公式解释**：
- **$W_{1,ij}$**：第一层权重矩阵的第$(i,j)$个元素
- **$W_{2,ij}$**：第二层权重矩阵的第$(i,j)$个元素
- **$\sum_{i,j} W_{1,ij}^2$**：第一层所有权重的平方和
- **$\sum_{i,j} W_{2,ij}^2$**：第二层所有权重的平方和
- **$\lambda$**：正则化强度（超参数），控制正则化项的权重
- **作用**：惩罚权重大大，防止过拟合，提高泛化能力

**总损失**：
$$L = L_{data} + R(W) = \frac{1}{N}\sum_{i=1}^{N} \left(-\log(p_{y_i})\right) + \lambda \left(\sum_{i,j} W_{1,ij}^2 + \sum_{i,j} W_{2,ij}^2\right)$$

**公式解释**：
- **$L$**：总损失，数据损失和正则化损失的和
- **目标**：最小化总损失，既要拟合数据（$L_{data}$小），又要防止过拟合（$R(W)$小）
- **平衡**：通过$\lambda$控制数据拟合和正则化的平衡

**设计原因**：交叉熵损失衡量预测与真实分布差异，L2正则化防止权重大大（这是验证集调优的基础），计算softmax时减去最大值保证数值稳定。

### 1.3 反向传播设计

**目标**：高效计算所有参数的梯度（这是解决梯度计算复杂问题的关键）。

**梯度计算**（链式法则）：

**1. Softmax损失梯度**：
$$\frac{\partial L}{\partial scores} = p - \mathbf{1}_{y_i}$$

**公式解释**：
- **$p$**：softmax概率矩阵，形状$(N, C)$，$p[i,j]$是样本$i$在类别$j$的概率
- **$\mathbf{1}_{y_i}$**：one-hot编码矩阵，形状$(N, C)$，$\mathbf{1}_{y_i}[i,y_i] = 1$，其他为0
- **$\frac{\partial L}{\partial scores}$**：损失对得分的梯度，形状$(N, C)$
- **计算过程**：
  - 对样本$i$，正确类别$y_i$：$\frac{\partial L}{\partial scores[i,y_i]} = p[i,y_i] - 1$
  - 对样本$i$，错误类别$j \neq y_i$：$\frac{\partial L}{\partial scores[i,j]} = p[i,j] - 0 = p[i,j]$
- **意义**：梯度表示预测概率与真实标签的差异，用于更新权重

**2. 第二层梯度**：

**权重梯度**：
$$\frac{\partial L}{\partial W_2} = a_1^T \cdot \frac{\partial L}{\partial scores} + 2\lambda W_2$$

**公式解释**：
- **$a_1^T$**：第一层激活的转置，形状$(H, N)$
- **$\frac{\partial L}{\partial scores}$**：损失对得分的梯度，形状$(N, C)$
- **$a_1^T \cdot \frac{\partial L}{\partial scores}$**：矩阵乘法，形状$(H, C)$，计算权重梯度
- **$2\lambda W_2$**：正则化项的梯度，$2\lambda$来自$\frac{\partial}{\partial W_2}(\lambda \sum W_{2,ij}^2) = 2\lambda W_2$
- **$\frac{\partial L}{\partial W_2}$**：损失对第二层权重的梯度，形状$(H, C)$
- **计算过程**：对每个权重$W_2[h,c]$，梯度来自所有样本的贡献：$\sum_{i=1}^{N} a_1[i,h] \cdot \frac{\partial L}{\partial scores[i,c]}$

**偏置梯度**：
$$\frac{\partial L}{\partial b_2} = \sum_{i=1}^{N} \frac{\partial L}{\partial scores_i}$$

**公式解释**：
- **$\frac{\partial L}{\partial scores_i}$**：样本$i$的得分梯度，形状$(C,)$
- **$\sum_{i=1}^{N}$**：对所有样本求和，形状$(C,)$
- **$\frac{\partial L}{\partial b_2}$**：损失对第二层偏置的梯度，形状$(C,)$
- **计算过程**：对每个类别$c$，$b_2[c]$的梯度是所有样本在该类别上得分梯度的和

**3. ReLU梯度**：
$$\frac{\partial a_1}{\partial z_1} = \mathbf{1}_{z_1 > 0}$$

**公式解释**：
- **$\mathbf{1}_{z_1 > 0}$**：指示函数矩阵，形状$(N, H)$
  - 当$z_1[i,j] > 0$时，$\mathbf{1}_{z_1 > 0}[i,j] = 1$
  - 当$z_1[i,j] \leq 0$时，$\mathbf{1}_{z_1 > 0}[i,j] = 0$
- **$\frac{\partial a_1}{\partial z_1}$**：ReLU对输入的梯度，形状$(N, H)$
- **意义**：ReLU在正区域梯度为1，负区域梯度为0，简单高效

**4. 第一层梯度**：

**激活梯度**：
$$\frac{\partial L}{\partial a_1} = \frac{\partial L}{\partial scores} \cdot W_2^T$$

**公式解释**：
- **$\frac{\partial L}{\partial scores}$**：损失对得分的梯度，形状$(N, C)$
- **$W_2^T$**：第二层权重的转置，形状$(C, H)$
- **$\frac{\partial L}{\partial a_1}$**：损失对第一层激活的梯度，形状$(N, H)$
- **计算过程**：通过链式法则，损失对激活的梯度 = 损失对得分的梯度 × 得分对激活的梯度

**线性输出梯度**：
$$\frac{\partial L}{\partial z_1} = \frac{\partial L}{\partial a_1} \odot \mathbf{1}_{z_1 > 0}$$

**公式解释**：
- **$\odot$**：逐元素相乘（Hadamard积）
- **$\frac{\partial L}{\partial a_1}$**：损失对激活的梯度，形状$(N, H)$
- **$\mathbf{1}_{z_1 > 0}$**：ReLU梯度，形状$(N, H)$
- **$\frac{\partial L}{\partial z_1}$**：损失对第一层线性输出的梯度，形状$(N, H)$
- **计算过程**：通过链式法则，损失对线性输出的梯度 = 损失对激活的梯度 × ReLU梯度

**权重梯度**：
$$\frac{\partial L}{\partial W_1} = X^T \cdot \frac{\partial L}{\partial z_1} + 2\lambda W_1$$

**公式解释**：
- **$X^T$**：输入数据的转置，形状$(D, N)$
- **$\frac{\partial L}{\partial z_1}$**：损失对第一层线性输出的梯度，形状$(N, H)$
- **$X^T \cdot \frac{\partial L}{\partial z_1}$**：矩阵乘法，形状$(D, H)$，计算权重梯度
- **$2\lambda W_1$**：正则化项的梯度
- **$\frac{\partial L}{\partial W_1}$**：损失对第一层权重的梯度，形状$(D, H)$

**偏置梯度**：
$$\frac{\partial L}{\partial b_1} = \sum_{i=1}^{N} \frac{\partial L}{\partial z_{1,i}}$$

**公式解释**：
- **$\frac{\partial L}{\partial z_{1,i}}$**：样本$i$的线性输出梯度，形状$(H,)$
- **$\sum_{i=1}^{N}$**：对所有样本求和，形状$(H,)$
- **$\frac{\partial L}{\partial b_1}$**：损失对第一层偏置的梯度，形状$(H,)$

**设计原因**：链式法则逐层计算梯度（一次前向+反向传播计算所有参数梯度，高效准确），ReLU梯度简单（正区域1，负区域0），矩阵运算批量计算提高效率。

**实现代码**：
```python
# Gradient from softmax loss
dscores = probs.copy()
dscores[np.arange(N), y] -= 1
dscores /= N

# Backprop through second layer
grads['W2'] = a1.T.dot(dscores) + 2 * reg * W2
grads['b2'] = np.sum(dscores, axis=0)

# Backprop through ReLU
da1 = dscores.dot(W2.T)
dz1 = da1.copy()
dz1[z1 <= 0] = 0  # ReLU gradient

# Backprop through first layer
grads['W1'] = X.T.dot(dz1) + 2 * reg * W1
grads['b1'] = np.sum(dz1, axis=0)
```

**关键点**：向量化矩阵运算代替循环，缓存中间结果（$z_1, a_1$）供反向传播复用，偏置梯度累加所有样本。

## 二、训练机制：SGD与超参数调优

基于核心逻辑，训练机制需要实现SGD高效优化和验证集调优。下面分别说明设计思路。

### 2.1 随机梯度下降（SGD）

**目标**：高效优化损失函数（这是SGD高效优化的核心机制）。

**实现逻辑**：随机采样batch，计算损失和梯度，更新权重，每个epoch衰减学习率。

**权重更新公式**：
$$W_1 \leftarrow W_1 - \alpha \cdot \frac{\partial L}{\partial W_1}$$
$$b_1 \leftarrow b_1 - \alpha \cdot \frac{\partial L}{\partial b_1}$$
$$W_2 \leftarrow W_2 - \alpha \cdot \frac{\partial L}{\partial W_2}$$
$$b_2 \leftarrow b_2 - \alpha \cdot \frac{\partial L}{\partial b_2}$$

**公式解释**：
- **$\alpha$**：学习率（learning_rate），控制更新步长
- **$\frac{\partial L}{\partial W_1}$**：损失对权重的梯度，形状与$W_1$相同
- **$W_1 - \alpha \cdot \frac{\partial L}{\partial W_1}$**：梯度下降更新，沿梯度反方向更新权重（因为要最小化损失）
- **学习率衰减**：每个epoch后，$\alpha \leftarrow \alpha \times \text{decay}$（如0.95），使后期更新更精细

**设计原因**：batch梯度计算比全批量快（适合大规模数据），随机采样跳出局部最优，学习率衰减后期精细优化。

**实现代码**：
```python
for it in range(num_iters):
    # 随机采样batch
    batch_indices = np.random.choice(num_train, batch_size, replace=True)
    X_batch = X[batch_indices]
    y_batch = y[batch_indices]
    
    # 计算损失和梯度
    loss, grads = self.loss(X_batch, y=y_batch, reg=reg)
    
    # 更新权重
    self.params['W1'] -= learning_rate * grads['W1']
    self.params['b1'] -= learning_rate * grads['b1']
    self.params['W2'] -= learning_rate * grads['W2']
    self.params['b2'] -= learning_rate * grads['b2']
    
    # 每个epoch衰减学习率
    if it % iterations_per_epoch == 0:
        learning_rate *= learning_rate_decay
```

### 2.2 超参数调优

**目标**：选择最优超参数组合（这是验证集调优的核心机制）。

**调优策略**：网格搜索在验证集上搜索最优超参数组合（隐藏层大小、学习率、正则化强度），保存最佳模型。

**网格搜索的具体过程**：

**步骤1：定义搜索空间**

首先，我们需要为每个超参数定义一个候选值列表：

```python
hidden_sizes = [100, 200, 300, 500]          # 4个候选值
learning_rates = [1e-3, 5e-4, 1e-4]         # 3个候选值
regularization_strengths = [0.05, 0.1, 0.25] # 3个候选值
```

**搜索空间大小**：4 × 3 × 3 = **36种组合**

**步骤2：遍历所有组合**

使用三重循环遍历所有超参数组合：

```python
best_val_acc = -1  # 记录最佳验证集准确率
best_net = None     # 保存最佳模型

for hidden_size in hidden_sizes:           # 外层循环：隐藏层大小
    for lr in learning_rates:              # 中层循环：学习率
        for reg in regularization_strengths: # 内层循环：正则化强度
            # 对每个组合执行以下操作
```

**遍历顺序示例**：
- 组合1：hidden_size=100, lr=1e-3, reg=0.05
- 组合2：hidden_size=100, lr=1e-3, reg=0.1
- 组合3：hidden_size=100, lr=1e-3, reg=0.25
- 组合4：hidden_size=100, lr=5e-4, reg=0.05
- ...（共36种组合）

**步骤3：对每个组合训练模型**

对每个超参数组合，执行以下操作：

```python
# 3.1 创建新网络（使用当前超参数）
net = TwoLayerNet(input_size, hidden_size, num_classes)
# 注意：每次创建新网络，权重随机初始化

# 3.2 在训练集上训练模型
stats = net.train(X_train, y_train, X_val, y_val,
                num_iters=3000,        # 训练3000次迭代
                batch_size=200,        # batch大小固定
                learning_rate=lr,      # 使用当前循环的学习率
                learning_rate_decay=0.95,  # 学习率衰减率固定
                reg=reg,               # 使用当前循环的正则化强度
                verbose=False)         # 不打印训练过程
```

**训练过程**：
- 使用训练集$X_{train}$和$y_{train}$训练模型
- 训练过程中，每个epoch在验证集$X_{val}$上评估准确率（用于监控，但不用于选择）
- 训练3000次迭代后，模型权重已更新

**步骤4：在验证集上评估**

训练完成后，在验证集上评估模型性能：

```python
# 4.1 使用训练好的模型预测验证集
y_val_pred = net.predict(X_val)  # 预测验证集标签

# 4.2 计算验证集准确率
val_acc = (y_val_pred == y_val).mean()
# 等价于：val_acc = np.mean(y_val_pred == y_val)
# 计算预测正确的样本比例
```

**评估指标**：验证集准确率（validation accuracy）

**步骤5：保存最佳模型**

比较当前组合的验证集准确率与历史最佳：

```python
# 5.1 比较验证集准确率
if val_acc > best_val_acc:
    # 如果当前组合更好，更新最佳记录
    best_val_acc = val_acc      # 更新最佳验证集准确率
    best_net = net              # 保存最佳模型（保存整个网络对象）
    
    # 可选：打印当前最佳组合
    print(f'New best: hidden_size={hidden_size}, lr={lr}, reg={reg}, val_acc={val_acc:.4f}')
```

**保存内容**：保存整个网络对象`best_net`，包括：
- 最佳超参数组合下的权重矩阵（$W_1, b_1, W_2, b_2$）
- 网络结构（隐藏层大小等）

**步骤6：完成搜索**

所有组合遍历完成后：

```python
print(f'Best validation accuracy: {best_val_acc:.4f}')
# 输出：Best validation accuracy: 0.5200（示例）

# best_net 就是最优超参数组合下的模型
# 可以在测试集上评估：test_acc = (best_net.predict(X_test) == y_test).mean()
```

**完整实现代码**：
```python
best_val_acc = -1
best_net = None

# 定义搜索空间
hidden_sizes = [100, 200, 300, 500]
learning_rates = [1e-3, 5e-4, 1e-4]
regularization_strengths = [0.05, 0.1, 0.25]

# 遍历所有组合（36种）
for hidden_size in hidden_sizes:
    for lr in learning_rates:
        for reg in regularization_strengths:
            # 创建网络
            net = TwoLayerNet(input_size, hidden_size, num_classes)
            
            # 训练（在训练集上）
            stats = net.train(X_train, y_train, X_val, y_val,
                            num_iters=3000, batch_size=200,
                            learning_rate=lr, learning_rate_decay=0.95,
                            reg=reg, verbose=False)
            
            # 评估（在验证集上）
            val_acc = (net.predict(X_val) == y_val).mean()
            
            # 更新最佳模型
            if val_acc > best_val_acc:
                best_val_acc = val_acc
                best_net = net
                
            # 打印当前组合结果（可选）
            print(f'hidden_size={hidden_size}, lr={lr}, reg={reg}, val_acc={val_acc:.4f}')

print(f'Best validation accuracy: {best_val_acc:.4f}')
```

**为什么使用验证集而不是测试集？**

1. **避免过拟合测试集**：如果直接在测试集上选择超参数，模型会"记住"测试集的特征，导致测试集准确率虚高，但泛化能力差
2. **模拟真实场景**：验证集模拟"未见过的新数据"，选择在验证集上表现最好的超参数，更可能在新数据上表现好
3. **数据划分**：
   - **训练集**（49000样本）：用于训练模型，更新权重
   - **验证集**（1000样本）：用于选择超参数，不参与训练
   - **测试集**（1000样本）：用于最终评估，只在最后使用一次

**设计原因**：网格搜索遍历所有组合找最优，验证集评估避免过拟合测试集（确保选择的超参数能够泛化到新数据）。

## 三、关键参数与效果

基于核心逻辑和训练机制，关键参数的选择直接影响最终效果。下面说明各参数的作用和影响。

### 3.1 关键参数

**隐藏层大小**：控制模型容量，决定网络的表达能力。太小（100）准确率50-53%，合适（300）准确率最高55%，太大（500）准确率50-54%未超过300。实际最优值：`300`。

**学习率**：控制更新步长。太大不稳定，太小（1e-4）训练慢准确率39-41%，合适（1e-3）准确率50-55%表现最好。实际最优值：`1e-3`。

**正则化强度**：防止过拟合。太小过拟合，太大（0.25）可能限制容量，合适（0.05）准确率51-55%表现最好。实际最优值：`0.05`。

**学习率衰减率**：每个epoch衰减因子。太大（0.9）过早停止，太小（0.99）不稳定，合适（0.95）平衡。典型值：`0.95`。

### 3.2 效果分析

**默认参数效果**：hidden_size=50时验证集准确率28.7%，模型容量不足，说明需要调优。

**调优后效果**：通过超参数调优（36种组合，最优：hidden_size=300, lr=1e-3, reg=0.05，迭代3000次），实际达到验证集55%，测试集54.7%，相比线性分类器（SVM 37-39%，Softmax 33.8%）显著提升（+15-20个百分点），证明非线性模型优势。

## 四、总结

**核心结论**：两层全连接神经网络通过非线性激活和反向传播实现复杂分类，准确率显著提升（54.7% vs 线性分类器37-39%）。

**核心机制**：ReLU激活引入非线性，反向传播高效计算梯度，SGD随机采样batch训练，验证集调优搜索最优超参数。四个机制协同工作，实现高效准确分类。

**关键优势**：非线性决策边界，高效梯度计算，强表达能力。

**性能表现**：默认参数（hidden_size=50）验证集28.7%，调优后实际54.7%（测试集），验证集55%，最优超参数（hidden_size=300, lr=1e-3, reg=0.05），SGD训练快，测试集与验证集准确率接近说明泛化良好。

**适用场景**：需要非线性决策边界的分类问题，中等规模数据集，高准确率要求。

**未来改进**：更深网络、其他激活函数、Dropout、批量归一化、卷积神经网络。
