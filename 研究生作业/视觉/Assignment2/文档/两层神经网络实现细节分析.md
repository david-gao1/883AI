## 一、训练模型实现：从数据到预测的完整流程

### 1.1 训练模型的整体架构

**训练目标**：学习权重矩阵$W_1, b_1, W_2, b_2$，使得模型能够准确分类图像。

**训练流程**：
```
输入数据 → 前向传播 → 计算损失 → 反向传播 → 更新权重 → 重复迭代
```

**实现代码结构**：
```python

```

### 1.2 为什么这样实现？

**1. 模块化设计**：
- **`loss`方法**：封装前向传播、损失计算、反向传播，可复用
- **`train`方法**：封装训练循环，处理SGD、学习率衰减等
- **`predict`方法**：封装预测逻辑，简单高效

**2. 参数管理**：
- 使用字典`self.params`统一管理所有参数，便于访问和更新
- 初始化时使用小随机值（`std=1e-4`），避免对称性问题和梯度消失

**3. 训练与预测分离**：
- 训练时使用`loss`方法（需要标签计算损失和梯度）
- 预测时只使用前向传播（不需要标签，只需要计算得分）

## 二、前向传播：从输入到得分的计算过程

### 2.1 前向传播的数学原理

**目标**：计算所有样本在所有类别上的得分。

**计算步骤**：

**步骤1：第一层线性变换**
$$z_1 = X \cdot W_1 + b_1$$

**具体计算**：
- 输入：$X$形状$(N, 3072)$，$N$个样本，每个样本3072个像素值
- 权重：$W_1$形状$(3072, H)$，$H$个隐藏单元，每个单元有3072个权重
- 输出：$z_1$形状$(N, H)$，每个样本的$H$个隐藏特征

**为什么需要这一步？**
- **特征提取**：将3072个原始像素值转换为$H$个更有用的特征
- **维度变换**：从高维（3072）降到低维（$H$，如300），提取关键信息
- **学习组合**：权重$W_1$学习如何组合像素值，提取边缘、纹理等模式

**步骤2：ReLU激活**
$$a_1 = \text{ReLU}(z_1) = \max(0, z_1)$$

**具体计算**：
- 输入：$z_1$形状$(N, H)$
- 输出：$a_1$形状$(N, H)$，与$z_1$相同
- 操作：逐元素取最大值，负值置为0

**为什么需要ReLU？**
- **引入非线性**：如果没有ReLU，多层线性变换等价于单层，无法学习非线性决策边界
- **激活特性**：只保留正值，抑制负值，相当于"激活"某些特征，"抑制"其他特征
- **稀疏性**：产生稀疏表示，有助于特征选择

**步骤3：第二层线性变换**
$$scores = a_1 \cdot W_2 + b_2$$

**具体计算**：
- 输入：$a_1$形状$(N, H)$，$H$个隐藏特征
- 权重：$W_2$形状$(H, 10)$，10个类别，每个类别有$H$个权重
- 输出：$scores$形状$(N, 10)$，每个样本在10个类别上的得分

**为什么需要这一步？**
- **特征到类别映射**：将$H$个隐藏特征映射到10个类别得分
- **分类决策**：得分最高的类别就是预测类别
- **学习权重组合**：权重$W_2$学习如何组合隐藏特征得到每个类别的得分

### 2.2 前向传播的实现代码

```python
def loss(self, X, y=None, reg=0.0):
    W1, b1 = self.params['W1'], self.params['b1']
    W2, b2 = self.params['W2'], self.params['b2']
    N, D = X.shape
    
    # 步骤1：第一层线性变换
    z1 = X.dot(W1) + b1  # (N, H)
    # 矩阵乘法：X (N, 3072) × W1 (3072, H) = z1 (N, H)
    # 偏置b1 (H,) 广播到 (N, H)
    
    # 步骤2：ReLU激活
    a1 = np.maximum(0, z1)  # (N, H)
    # 逐元素取最大值：a1[i,j] = max(0, z1[i,j])
    
    # 步骤3：第二层线性变换
    scores = a1.dot(W2) + b2  # (N, C)
    # 矩阵乘法：a1 (N, H) × W2 (H, 10) = scores (N, 10)
    # 偏置b2 (10,) 广播到 (N, 10)
    
    # 如果只需要得分（预测时），返回scores
    if y is None:
        return scores
    
    # 如果需要计算损失和梯度（训练时），继续...
```

**关键实现细节**：

1. **矩阵乘法**：使用`X.dot(W1)`一次性计算所有样本的所有特征，比循环快得多
2. **广播机制**：偏置`b1`形状$(H,)$自动广播到$(N, H)$，无需显式复制
3. **缓存中间结果**：保存`z1`和`a1`，供反向传播使用

## 三、反向传播：从损失到梯度的计算过程

### 3.1 反向传播的数学原理

**目标**：计算损失对所有参数的梯度，用于更新权重。

**核心思想**：链式法则（Chain Rule）

**计算步骤**（从输出层向输入层逐层计算）：

**步骤1：Softmax损失梯度**

$$L = \frac{1}{N}\sum_i \left(-\log(p_{y_i})\right) + \lambda R(W)$$

$$\frac{\partial L}{\partial scores} = p - \mathbf{1}_{y_i}$$

**具体计算**：
```python
# 计算softmax概率
exp_scores = np.exp(scores - np.max(scores, axis=1, keepdims=True))
probs = exp_scores / np.sum(exp_scores, axis=1, keepdims=True)

# 计算损失梯度
dscores = probs.copy()  # (N, C)
dscores[np.arange(N), y] -= 1  # 正确类别位置减去1
dscores /= N  # 平均
```

**为什么这样计算？**
- **概率与标签的差异**：梯度表示预测概率与真实标签的差异
- **正确类别**：$p_{y_i} - 1$，鼓励提高正确类别概率
- **错误类别**：$p_j$，鼓励降低错误类别概率

**步骤2：第二层梯度**

$$\frac{\partial L}{\partial W_2} = a_1^T \cdot \frac{\partial L}{\partial scores} + 2\lambda W_2$$

$$\frac{\partial L}{\partial b_2} = \sum_{i=1}^{N} \frac{\partial L}{\partial scores_i}$$

**具体计算**：
```python
# 权重梯度
grads['W2'] = a1.T.dot(dscores) + 2 * reg * W2
# a1.T (H, N) × dscores (N, C) = (H, C)
# 加上正则化项梯度：2 * reg * W2

# 偏置梯度
grads['b2'] = np.sum(dscores, axis=0)
# 对所有样本求和：(C,)
```

**为什么这样计算？**
- **链式法则**：$\frac{\partial L}{\partial W_2} = \frac{\partial L}{\partial scores} \cdot \frac{\partial scores}{\partial W_2} = \frac{\partial L}{\partial scores} \cdot a_1$
- **矩阵转置**：$a_1^T$用于匹配维度，实现批量计算
- **正则化项**：$2\lambda W_2$来自$\frac{\partial}{\partial W_2}(\lambda \sum W_{2,ij}^2)$

**步骤3：ReLU梯度**

$$\frac{\partial a_1}{\partial z_1} = \mathbf{1}_{z_1 > 0}$$

**具体计算**：
```python
# 计算损失对激活的梯度
da1 = dscores.dot(W2.T)  # (N, H)
# dscores (N, C) × W2.T (C, H) = (N, H)

# ReLU梯度：正区域1，负区域0
dz1 = da1.copy()
dz1[z1 <= 0] = 0  # 负区域梯度为0
```

**为什么这样计算？**
- **ReLU特性**：在正区域梯度为1，负区域梯度为0
- **链式法则**：$\frac{\partial L}{\partial z_1} = \frac{\partial L}{\partial a_1} \cdot \frac{\partial a_1}{\partial z_1}$
- **逐元素相乘**：使用布尔索引`z1 <= 0`选择负区域，置为0

**步骤4：第一层梯度**

$$\frac{\partial L}{\partial W_1} = X^T \cdot \frac{\partial L}{\partial z_1} + 2\lambda W_1$$

$$\frac{\partial L}{\partial b_1} = \sum_{i=1}^{N} \frac{\partial L}{\partial z_{1,i}}$$

**具体计算**：
```python
# 权重梯度
grads['W1'] = X.T.dot(dz1) + 2 * reg * W1
# X.T (3072, N) × dz1 (N, H) = (3072, H)
# 加上正则化项梯度

# 偏置梯度
grads['b1'] = np.sum(dz1, axis=0)
# 对所有样本求和：(H,)
```

### 3.2 反向传播的实现代码

```python
def loss(self, X, y=None, reg=0.0):
    # ... 前向传播代码 ...
    
    # 计算损失
    exp_scores = np.exp(scores - np.max(scores, axis=1, keepdims=True))
    probs = exp_scores / np.sum(exp_scores, axis=1, keepdims=True)
    correct_logprobs = -np.log(probs[np.arange(N), y])
    data_loss = np.sum(correct_logprobs) / N
    reg_loss = reg * (np.sum(W1 * W1) + np.sum(W2 * W2))
    loss = data_loss + reg_loss
    
    # 反向传播
    grads = {}
    
    # 步骤1：Softmax损失梯度
    dscores = probs.copy()
    dscores[np.arange(N), y] -= 1
    dscores /= N
    
    # 步骤2：第二层梯度
    grads['W2'] = a1.T.dot(dscores) + 2 * reg * W2
    grads['b2'] = np.sum(dscores, axis=0)
    
    # 步骤3：ReLU梯度
    da1 = dscores.dot(W2.T)
    dz1 = da1.copy()
    dz1[z1 <= 0] = 0
    
    # 步骤4：第一层梯度
    grads['W1'] = X.T.dot(dz1) + 2 * reg * W1
    grads['b1'] = np.sum(dz1, axis=0)
    
    return loss, grads
```

**关键实现细节**：

1. **链式法则应用**：从输出层向输入层逐层计算，每层使用上一层的梯度
2. **矩阵运算**：使用矩阵乘法批量计算所有样本的梯度，高效
3. **缓存复用**：使用前向传播时保存的`z1`和`a1`，避免重复计算
4. **正则化项**：在梯度中加入正则化项梯度，防止权重大大

## 四、训练循环：SGD与学习率衰减

### 4.1 训练循环的实现

**目标**：通过多次迭代更新权重，最小化损失函数。

**实现代码**：
```python
def train(self, X, y, X_val, y_val,
          learning_rate=1e-3, learning_rate_decay=0.95,
          reg=5e-6, num_iters=100,
          batch_size=200, verbose=False):
    
    num_train = X.shape[0]
    iterations_per_epoch = max(num_train / batch_size, 1)
    
    loss_history = []
    train_acc_history = []
    val_acc_history = []
    
    for it in range(num_iters):
        # 步骤1：随机采样batch
        batch_indices = np.random.choice(num_train, batch_size, replace=True)
        X_batch = X[batch_indices]
        y_batch = y[batch_indices]
        
        # 步骤2：计算损失和梯度
        loss, grads = self.loss(X_batch, y=y_batch, reg=reg)
        loss_history.append(loss)
        
        # 步骤3：更新权重
        self.params['W1'] -= learning_rate * grads['W1']
        self.params['b1'] -= learning_rate * grads['b1']
        self.params['W2'] -= learning_rate * grads['W2']
        self.params['b2'] -= learning_rate * grads['b2']
        
        # 步骤4：每个epoch衰减学习率
        if it % iterations_per_epoch == 0:
            learning_rate *= learning_rate_decay
            
            # 步骤5：评估准确率（可选）
            train_acc = (self.predict(X_batch) == y_batch).mean()
            val_acc = (self.predict(X_val) == y_val).mean()
            train_acc_history.append(train_acc)
            val_acc_history.append(val_acc)
    
    return {
        'loss_history': loss_history,
        'train_acc_history': train_acc_history,
        'val_acc_history': val_acc_history,
    }
```

### 4.2 为什么这样实现？

**1. 随机采样batch**：
- **效率**：每次只计算batch的梯度，比全批量梯度下降快
- **随机性**：随机采样有助于跳出局部最优
- **内存**：只需存储batch数据，内存占用小

**2. 梯度下降更新**：
- **更新规则**：$W \leftarrow W - \alpha \cdot \frac{\partial L}{\partial W}$
- **学习率**：$\alpha$控制更新步长，太大不稳定，太小训练慢
- **负号**：沿梯度反方向更新，因为要最小化损失

**3. 学习率衰减**：
- **目的**：训练初期用较大学习率快速收敛，后期用较小学习率精细优化
- **实现**：每个epoch后乘以衰减率（如0.95）
- **效果**：帮助模型收敛到更好的局部最优

**4. 准确率监控**：
- **目的**：监控训练过程，判断是否过拟合
- **训练集准确率**：判断模型是否拟合数据
- **验证集准确率**：判断模型是否泛化良好

## 五、网格搜索：在验证集上搜索最优超参数

### 5.1 网格搜索的实现逻辑

**目标**：在验证集上搜索最优超参数组合，避免过拟合测试集。

**搜索空间定义**：
```python
hidden_sizes = [100, 200, 300, 500]          # 4个候选值
learning_rates = [1e-3, 5e-4, 1e-4]         # 3个候选值
regularization_strengths = [0.05, 0.1, 0.25] # 3个候选值
```

**搜索空间大小**：4 × 3 × 3 = **36种组合**

**实现代码**：
```python
best_val_acc = -1
best_net = None

# 三重循环遍历所有组合
for hidden_size in hidden_sizes:
    for lr in learning_rates:
        for reg in regularization_strengths:
            # 步骤1：创建新网络（每次重新初始化）
            net = TwoLayerNet(input_size, hidden_size, num_classes)
            
            # 步骤2：在训练集上训练
            stats = net.train(X_train, y_train, X_val, y_val,
                            num_iters=3000, batch_size=200,
                            learning_rate=lr, learning_rate_decay=0.95,
                            reg=reg, verbose=False)
            
            # 步骤3：在验证集上评估
            val_acc = (net.predict(X_val) == y_val).mean()
            
            # 步骤4：更新最佳模型
            if val_acc > best_val_acc:
                best_val_acc = val_acc
                best_net = net
            
            # 打印当前组合结果
            print(f'hidden_size={hidden_size}, lr={lr}, reg={reg}, val_acc={val_acc:.2f}')

print(f'Best validation accuracy: {best_val_acc:.2f}')
```

### 5.2 为什么使用验证集？

**数据划分**：
- **训练集**（49000样本）：用于训练模型，更新权重
- **验证集**（1000样本）：用于选择超参数，不参与训练
- **测试集**（1000样本）：用于最终评估，只在最后使用一次

**为什么不用测试集选择超参数？**
- **避免过拟合**：如果在测试集上选择超参数，模型会"记住"测试集的特征
- **泛化能力**：验证集模拟"未见过的新数据"，选择在验证集上表现最好的超参数，更可能在新数据上表现好
- **科学评估**：测试集只用于最终评估，确保评估结果可靠

### 5.3 网格搜索的优缺点

**优点**：
- **全面**：遍历所有组合，不会遗漏最优解
- **简单**：实现简单，易于理解
- **可重复**：结果可重复，便于调试

**缺点**：
- **计算量大**：36种组合，每种训练3000次迭代，总计算量大
- **效率低**：如果搜索空间大，计算时间很长
- **固定网格**：只能搜索预定义的候选值，可能错过最优值

**改进方向**：
- **随机搜索**：随机采样超参数组合，可能更快找到最优解
- **贝叶斯优化**：使用概率模型指导搜索，更高效
- **粗到细搜索**：先粗粒度搜索，再在最优区域细粒度搜索

## 六、默认参数分析：为什么准确率只有28.7%？

### 6.1 默认参数设置

**默认参数**：
```python
hidden_size = 50
learning_rate = 1e-4
reg = 0.25
num_iters = 1000
batch_size = 200
learning_rate_decay = 0.95
```

**验证集准确率**：28.7%

### 6.2 各参数的意义和影响

**1. 隐藏层大小（hidden_size=50）**

**意义**：控制模型容量，决定网络能学习多少种特征模式。

**影响**：
- **太小（50）**：模型容量不足，只能学习少量特征模式，无法充分表达数据的复杂性
- **结果**：欠拟合，准确率低（28.7%）

**为什么50太小？**
- CIFAR-10有10个类别，每个类别有复杂的视觉特征（边缘、纹理、颜色等）
- 50个隐藏单元不足以学习所有这些特征模式
- 类比：50个"专家"无法覆盖10个复杂类别的所有特征

**2. 学习率（learning_rate=1e-4）**

**意义**：控制权重更新的步长。

**影响**：
- **太小（1e-4）**：每次更新步长很小，训练慢，可能未充分训练
- **结果**：1000次迭代后，模型可能还未收敛到最优

**为什么1e-4太小？**
- 从损失曲线看，损失从2.30降到1.95，下降缓慢
- 1000次迭代可能不够，需要更多迭代才能充分训练
- 或者需要更大的学习率加快收敛

**3. 正则化强度（reg=0.25）**

**意义**：防止过拟合，限制权重大小。

**影响**：
- **较大（0.25）**：较强的正则化，限制模型容量
- **结果**：与隐藏层大小50结合，进一步限制模型表达能力

**为什么0.25可能太大？**
- 当模型容量本身就不足（hidden_size=50）时，强正则化会进一步限制模型
- 应该先增加模型容量，再考虑正则化

**4. 迭代次数（num_iters=1000）**

**意义**：训练轮数，控制训练时间。

**影响**：
- **较少（1000）**：可能未充分训练
- **结果**：模型可能还未收敛

**为什么1000可能不够？**
- 从损失曲线看，损失仍在下降，说明模型还在学习
- 需要更多迭代才能充分训练

### 6.3 默认参数的问题总结

## 十、常见问题与调试技巧

### 10.1 损失不下降

**可能原因**：
1. 学习率太小：增加学习率
2. 初始化不当：检查权重初始化
3. 梯度计算错误：使用梯度检查验证

**调试方法**：
```python
# 梯度检查
from cs231n.gradient_check import eval_numerical_gradient
f = lambda W: net.loss(X, y, reg=0.05)[0]
grad_numerical = eval_numerical_gradient(f, net.params['W1'], verbose=False)
print('Relative error:', rel_error(grad_numerical, grads['W1']))
```

### 10.2 准确率不提升

**可能原因**：
1. 模型容量不足：增加隐藏层大小
2. 训练不充分：增加迭代次数
3. 学习率不合适：调整学习率

**调试方法**：
- 检查训练集准确率：如果训练集准确率低，说明模型容量不足
- 检查验证集准确率：如果验证集准确率低但训练集高，说明过拟合
- 检查损失曲线：如果损失不下降，说明学习率或初始化有问题

### 10.3 过拟合

**可能原因**：
1. 模型容量太大：减少隐藏层大小
2. 正则化太弱：增加正则化强度
3. 训练数据太少：增加训练数据

**调试方法**：
- 比较训练集和验证集准确率：如果差距大，说明过拟合
- 增加正则化强度：观察验证集准确率是否提升
- 减少模型容量：观察是否改善
